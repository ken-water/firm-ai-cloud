# CloudOps One v0.1.24

## Release Metadata

- Version: `v0.1.24`
- Release date: `2026-03-19`
- Release type: `minor`
- Git tag: `v0.1.24`
- GitHub issues: `#247, #248, #249, #250, #251`

## Executive Summary

v0.1.24 focuses on customer-demo readiness. This release improves module-first navigation clarity, visual cockpit hierarchy, and topology readability while adding a repeatable demo scenario pack.

## Highlights

- Module cockpit now has a structured entry map: `Observe -> Operate -> Assets`.
- KPI first-glance strip surfaces key risk counts for fast operator scanning.
- Topology board now includes lane grouping (`critical/warning/healthy`) and owner-state hints.
- Demo scenario automation script provides reusable validation artifacts for walkthrough evidence.

## Changed

- Module operation cockpit now exposes:
  - module entry map for task-oriented navigation,
  - KPI first-glance strip,
  - lane-based topology readability panel.
- Demo flow guidance is now explicit and repeatable through scenario checklist and automation script.

## Fixed

- Reduced customer-demo confusion caused by dense module entry and flat visual hierarchy.
- Improved topology readability for non-expert audiences without removing detailed drill paths.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.24` code and restart API/web services.
2. Ensure API and web-console are running with latest migrations.
3. Verify demo-oriented endpoints and views:
   - `GET /api/v1/ops/cockpit/topology-board`
   - `GET /api/v1/ops/cockpit/handover-readiness`
4. Optionally run demo validation script:
   - `bash scripts/qa-v0.1.24-demo-scenarios.sh --api-base-url http://127.0.0.1:8080 --auth-user admin`
5. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`
