# CloudOps One v0.1.23

## Release Metadata

- Version: `v0.1.23`
- Release date: `2026-03-18`
- Release type: `minor`
- Git tag: `v0.1.23`
- GitHub issues: `#241, #242, #243, #244, #245`

## Executive Summary

v0.1.23 upgrades cockpit decision flow from isolated read panels to an integrated operator path: copilot guided actions, topology board risk/owner view, and handoff readiness state model. The release keeps write actions explicitly guarded while giving SMB operators a faster evidence-to-action loop.

## Highlights

- AI evidence responses now include guided action contracts with safety boundaries.
- Topology board v1 provides deterministic service risk ordering with owner focus.
- Handoff readiness introduces explicit `ready / at_risk / blocking` states with explainable reasons.
- Cockpit now offers one-click integrated flow sync across copilot/topology/handoff modules.

## Changed

- AI evidence query contract now includes `guided_actions` with:
  - `action_type`,
  - `safety_boundary`,
  - `risk_level`,
  - `requires_approval`,
  - `requires_write`,
  - evidence refs/metrics.
- Topology risk panel now includes board cards with owner state, risk level, and recommended action.
- One-screen handoff section now includes readiness summary and prioritized readiness items.
- Cockpit includes integrated flow panel for copilot -> topology -> handoff synchronization.

## Fixed

- Reduced cross-panel context switching for operators when deciding shift actions.
- Guided write actions now display guard and approval expectations explicitly.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.23` code and restart API/web services.
2. Ensure API and web-console are running with latest migrations.
3. Verify new endpoints:
   - `GET /api/v1/ops/cockpit/topology-board`
   - `GET /api/v1/ops/cockpit/handover-readiness`
4. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`

## Known Issues

- Guided actions are safety-aware recommendations; full autonomous multi-step execution is still out of scope.
- Topology board owner attribution is baseline-derived and will be refined with richer org ownership models.
