# CloudOps One v0.0.2

## Release Metadata

- Version: `v0.0.2`
- Release date: `2026-03-02`
- Release type: `minor`
- Git tag: `v0.0.2`

## Executive Summary

`v0.0.2` delivers the first operational CMDB loop for relationship management and discovery-driven changes.

This release introduces dynamic CMDB extensibility, QR/barcode lookup, relationship graph APIs, multi-source discovery (including container-oriented seed mode), change event detection, and notification dispatch with retry and delivery logs.

## Highlights

- Dynamic CMDB fields with type/length/enum validation and scanner-ready metadata.
- Discovery pipeline upgraded to real execution mode with candidate review (approve/reject) and asset auto create/merge.
- Change detection now emits standardized events (`asset.new_detected`, `asset.profile_changed`, `asset.offboarded_suspected`).
- Notification management and dispatch are integrated, including webhook retry and delivery tracking.
- Web console now includes relation management and discovery review operations.

## Added

- CMDB schema and APIs:
  - Asset custom fields (`custom_fields`) and field definitions (`field-definitions`).
  - Asset code fields (`qr_code`, `barcode`) and scan lookup endpoint.
  - Asset relation table and APIs (`/relations`, `/assets/{id}/graph`).
- Discovery module:
  - Discovery jobs and candidates persistence.
  - Discovery job run endpoint with run stats.
  - Source adapters for:
    - `zabbix_hosts`
    - `snmp_seed`
    - `k8s_seed` (container-oriented seed input)
  - Candidate review APIs (`approve`, `reject`) with auto create/merge.
- Discovery eventing:
  - Event table and query API (`/discovery/events`).
  - Offboarded threshold support via `scope.offboarded_threshold`.
- Notification module:
  - Notification channels, templates, and subscriptions APIs.
  - Delivery log table and query API (`/discovery/notification-deliveries`).
  - Webhook retry with exponential backoff.
- Frontend:
  - CMDB relation panel with create/delete actions and summary counters.
  - Discovery jobs panel and candidate review actions.
- Testing/ops:
  - Integration smoke script: `scripts/test-cmdb-loop.sh`.
- Documentation:
  - Expanded developer quickstart with relation/discovery/notification flows.
  - Troubleshooting section for discovery, review, and delivery failures.

## Changed

- Discovery run endpoint now executes actual collection + candidate enqueue + change detection, instead of only queue-marking a job.
- Discovery event emission now triggers notification dispatch flow and delivery tracking.
- Web console upgraded from list-only CMDB view to operational CMDB + discovery workflow view.

## Fixed

- Improved conflict handling and validation error messages for unique QR/barcode and relation duplicates.
- Normalized source type handling across discovery adapters.

## Breaking Changes

- None

## Upgrade Steps

1. Pull latest code and rebuild:
   - `git pull origin main`
   - `cargo check --workspace`
2. Start dependency stack and run API so migrations apply automatically:
   - `bash scripts/install.sh --skip-docker-install --mirror cn`
   - `cargo run -p api`
3. (Optional) verify end-to-end behavior:
   - `bash scripts/test-cmdb-loop.sh`

## Validation

- Backend: `cargo check --workspace` passed.
- Frontend: `npm run build` in `apps/web-console` passed.
- Smoke tests: script `scripts/test-cmdb-loop.sh` added and shell syntax validated (`bash -n`).

## Known Issues

- Notification email channel is currently treated as baseline channel type; production SMTP delivery integration is not implemented yet.
- Container discovery is currently seed-driven (`k8s_seed`) and should be extended with direct Kubernetes API polling in a future release.
