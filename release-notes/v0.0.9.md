# CloudOps One v0.0.9

## Release Metadata

- Version: `v0.0.9`
- Release date: `2026-03-05`
- Release type: `minor`
- Git tag: `v0.0.9`

## Executive Summary

v0.0.9 delivers the first release-grade topology baseline (API + workspace), adds stream lag observability,
improves monitoring-sync burst handling, introduces benchmark threshold gating with optional CI integration,
and standardizes quarterly disaster-recovery drills with auditable report artifacts.

## Highlights

- Added topology map API and web topology workspace route (`#/topology`) with graph drill drawers.
- Added stream lag metrics endpoint and worker burst tuning knobs for reliability tracking.
- Added benchmark threshold gate script/policy and optional manual CI benchmark gate job.
- Added DR drill automation script that generates machine-readable and markdown reports.

## Added

- Topology API baseline:
  - `GET /api/v1/topology/maps/{scope}`
  - scope key parsing: `global`, `site:<value>`, `department:<value>`
  - deterministic node/edge window payload with `empty` marker and health normalization
- Topology workspace page:
  - route: `#/topology`
  - node and edge drill drawers with scope-aware quick actions
- Stream lag metrics API:
  - `GET /api/v1/streams/metrics`
  - lag summary: `samples`, `p50`, `p95`, `p99`, `max`
  - lag buckets for frontend-friendly trend visualization
- Benchmark quality gate:
  - policy file: `scripts/benchmark-threshold-policy.json`
  - gate script: `scripts/benchmark-threshold-gate.sh`
  - report artifacts: `gate-summary.json`, `gate-summary.md`
- DR drill automation:
  - script: `scripts/dr-drill.sh`
  - report artifacts: `report.json`, `report.md`, `logs/*.log`

## Changed

- Monitoring-sync worker burst path:
  - batch claim mode and bounded parallel processing
  - configurable env knobs:
    - `MONITORING_SYNC_BATCH_SIZE`
    - `MONITORING_SYNC_MAX_PARALLEL`
  - queue-depth and throughput telemetry logs per batch
- Manual CI workflow:
  - new optional benchmark gate job controlled by `workflow_dispatch` input `run_bench`
  - benchmark artifacts are uploaded and summarized in run output
- RBAC coverage:
  - explicit permission mapping and tests for `/api/v1/streams/metrics`
  - topology route mapping to `cmdb.relations.read`

## Fixed

- CI `auth-rbac-integration` startup stability:
  - build API binary before startup
  - run `target/debug/api` directly
  - fail fast when API process exits before readiness
- CI `auth-rbac-integration` OIDC dev script stability:
  - avoid subshell state loss when capturing HTTP status in `scripts/test-oidc-dev.sh`
  - preserve response body file path for logout revocation assertion

## Breaking Changes

- None.

## Upgrade Steps

1. Pull release code and install dependencies for `v0.0.9`.
2. Restart API service to apply new topology/stream/worker endpoints and worker env knobs.
3. Run backend and frontend verification:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`
4. Run benchmark and gate checks as needed:
   - `bash scripts/benchmark-api-load.sh --concurrency 10`
   - `bash scripts/benchmark-sse-burst-smoke.sh --burst-count 300`
   - `bash scripts/benchmark-threshold-gate.sh --api-summary <api-summary.csv> --sse-summary <sse-summary.json>`
5. Run a DR drill in non-production:
   - `bash scripts/dr-drill.sh --env-file deploy/.env --output-dir .run/dr-drill/<run-id> --yes`

## Known Issues

- Topology workspace currently uses deterministic ring layout; very dense edge sets can still cause visual overlap.
- Stream lag metrics are computed from recent sync-job windows and are intended as operational summary, not full historical analytics.
- DR drill script assumes local Docker/Compose reachability and non-production safety controls.
