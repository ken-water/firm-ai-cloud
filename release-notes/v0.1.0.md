# CloudOps One v0.1.0

## Release Metadata

- Version: `v0.1.0`
- Release date: `2026-03-05`
- Release type: `minor`
- Git tag: `v0.1.0`

## Executive Summary

v0.1.0 delivers the first operator-simplicity baseline for non-expert teams: a first-run setup wizard,
a unified alert center with acknowledge/close lifecycle controls, and alert-to-ticket automation templates.
This release is focused on reducing onboarding friction and making incident handling auditable from the UI.

## Highlights

- Added setup preflight and integration checklist APIs with remediation hints for each failed check.
- Added setup wizard page (`#/setup`) so first-time operators can complete initialization from the web console.
- Added unified alert center page (`#/alerts`) with filters, bulk acknowledge/close, and detail timeline drill panel.
- Added alert-to-ticket policy engine baseline with built-in templates and dedup window controls.

## Added

- Setup readiness APIs:
  - `GET /api/v1/setup/preflight`
  - `GET /api/v1/setup/checklist`
- Unified alert domain migration and APIs:
  - migration: `services/api/migrations/202603050003_create_alerting_and_setup_permissions.sql`
  - `GET /api/v1/alerts`
  - `GET /api/v1/alerts/{id}`
  - `POST /api/v1/alerts/{id}/ack`
  - `POST /api/v1/alerts/{id}/close`
  - `POST /api/v1/alerts/bulk/ack`
  - `POST /api/v1/alerts/bulk/close`
  - `GET/POST/PATCH /api/v1/alerts/policies*`
- Alert-to-ticket policy templates:
  - `critical-infrastructure`
  - `service-degradation`
  - `repeated-failure` (disabled by default)
- Web-console routes and pages:
  - `#/setup`
  - `#/alerts`
- New RBAC permissions:
  - `ops.setup.read`
  - `alerts.read`
  - `alerts.write`

## Changed

- Monitoring sync worker now opens/updates unified alerts on failures and auto-closes the same alert on recovery.
- Route permission mapping and RBAC matrix checks now include setup and alert center endpoints.
- Developer quickstart and RBAC coverage documentation now include setup and alert-center usage examples.
- Changelog now records a dedicated `0.0.9` section and a new `0.1.0` section.

## Fixed

- Removed unused alert status branch in API alert lifecycle handler to keep warning-free API test builds.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull the `v0.1.0` code and apply database migrations.
2. Restart API and web-console services so new setup/alert routes are available.
3. Verify setup checks and alert lifecycle endpoints with quickstart curl examples in `docs/07-dev-quickstart.md`.
4. Verify RBAC matrix includes `ops.setup.read` and `alerts.read/write` for expected roles.
5. Validate backend and frontend quality gates listed in the Validation section.

## Known Issues

- Setup wizard currently validates readiness and provides remediation hints, but does not execute auto-remediation actions.
- Alert center currently focuses on internal lifecycle handling and does not include external pager escalation integrations.
- Very large alert datasets may require further pagination UX refinement beyond the current baseline table flow.
