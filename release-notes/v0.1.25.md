## Executive Summary

v0.1.25 hardens customer-demo execution with explicit guided checkpoints, business-value KPI narratives, and a productized pilot feedback loop with closure status tracking. The release process now enforces demo acceptance artifact metadata for consistent governance.

## Highlights

- Module cockpit now includes ordered demo narrative checkpoints with evidence and direct action links.
- KPI cards now include deterministic threshold rules and business-outcome narratives.
- Setup activation now includes pilot feedback queue and closure status workflow (`open/in_progress/resolved`).
- Release issue gate now validates demo acceptance artifact metadata for `v0.1.25+`.

## Added

- `v0.1.25` planning and release-gate docs:
  - `docs/72-v0.1.25-demo-hardening-feedback-loop-plan.md`
  - `docs/73-v0.1.25-release-gate-checklist.md`
- Setup activation feedback APIs:
  - `GET /api/v1/setup/activation/feedback`
  - `POST /api/v1/setup/activation/feedback/{id}/closure`

## Changed

- Overview module cockpit now includes a deterministic demo narrative checkpoint panel.
- KPI first-glance cards now expose business value hints and threshold rules per metric.
- Setup activation feedback submission now writes structured context (`category`, `severity`, `module`, `expected_value`) for pilot analysis.
- Release gate script now enforces demo acceptance metadata and artifact existence for `v0.1.25+`.

## Fixed

- Demo walkthrough no longer depends on presenter memory to maintain ordered story flow.
- Pilot feedback closure status is now product-visible instead of ad-hoc follow-up.
- Release governance now fails fast when demo acceptance artifacts are missing from release metadata.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.25` code and restart API/web services.
2. Ensure setup activation feedback endpoints are reachable:
   - `GET /api/v1/setup/activation/feedback`
   - `POST /api/v1/setup/activation/feedback/{id}/closure`
3. Open setup wizard and verify `First-value activation` includes `Pilot feedback loop` panel.
4. Verify module cockpit shows `Demo narrative checkpoints` and KPI value narratives.
5. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`
