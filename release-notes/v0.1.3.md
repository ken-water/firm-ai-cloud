# CloudOps One v0.1.3

## Release Metadata

- Version: `v0.1.3`
- Release date: `2026-03-06`
- Release type: `minor`
- Git tag: `v0.1.3`

## Executive Summary

v0.1.3 completes the operator-simplicity and production-hardening closure for the current v0.1.x track.

The release adds no-code onboarding templates, one-click alert remediation with safety confirmation, auditable guided checklist tracking, and a reproducible `scale-10k` benchmark evidence path with CI regression guardrails.

## Highlights

- Added no-code setup bootstrap templates for identity, monitoring source, and notification baseline setup.
- Added mapped alert-to-playbook remediation flow with dry-run confirmation and timeline persistence.
- Added guided daily checklist tracking with per-operator/day persistence, overdue/skipped highlighting, and exception audit logging.
- Extended benchmark tooling to `scale-10k` and added CI trend regression guard with actionable impacted-metric summaries.

## Added

- Setup bootstrap template APIs:
  - `GET /api/v1/setup/templates`
  - `POST /api/v1/setup/templates/{key}/preview`
  - `POST /api/v1/setup/templates/{key}/apply`
- Alert remediation planning API:
  - `GET /api/v1/alerts/{id}/remediation`
- Cockpit checklist APIs:
  - `GET /api/v1/ops/cockpit/checklists`
  - `POST /api/v1/ops/cockpit/checklists/{template_key}/complete`
  - `POST /api/v1/ops/cockpit/checklists/{template_key}/exception`
- New migrations:
  - `services/api/migrations/202603060001_create_local_mfa_recovery_codes.sql`
  - `services/api/migrations/202603060002_create_setup_bootstrap_templates.sql`
  - `services/api/migrations/202603060003_create_ops_checklist_tables.sql`
- Benchmark regression guard assets:
  - `scripts/benchmark-regression-guard.sh`
  - `scripts/benchmark-regression-policy.json`
  - `benchmarks/baselines/scale-10k/api-summary.csv`
  - `benchmarks/baselines/scale-10k/sse-summary.json`
- Capacity guidance and reproducibility doc:
  - `docs/24-v0.1.3-capacity-guidance.md`

## Changed

- Web console alert center now supports one-click mapped remediation with confirmation-token flow for risky actions.
- Daily operations cockpit now includes guided checklist management and latest remediation context in queue rationale details.
- Benchmark profile scripts now support `scale-10k`:
  - `scripts/benchmark-api-load.sh`
  - `scripts/benchmark-sse-burst-smoke.sh`
  - `scripts/benchmark-scale-profiles.sh`
  - `scripts/benchmark-threshold-policy.json`
- CI benchmark job now runs profile gate + trend delta + regression guard and publishes profile/gate/trend/regression summaries.
- RBAC mapping coverage now includes cockpit checklist write routes (`ops.cockpit.write`) and integration checks.

## Fixed

- Shared required-setting validation message remains identity-provider-agnostic (`<KEY> is required`) across auth hardening paths.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.3` code and apply database migrations.
2. Restart API and web-console services.
3. Verify setup template preview/apply flow from `#/setup`.
4. Verify alert remediation flow from `#/alerts` including high-risk confirmation path.
5. Verify guided checklist read/write flow from `#/overview` daily cockpit section.
6. For scale validation, run `scale-10k` profile and regression guard using commands in `docs/24-v0.1.3-capacity-guidance.md`.
