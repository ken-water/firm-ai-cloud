# CloudOps One v0.0.4

## Release Metadata

- Version: `v0.0.4`
- Release date: `2026-03-03`
- Release type: `patch`
- Git tag: `v0.0.4`

## Executive Summary

`v0.0.4` delivers the first full UI hardening iteration for CloudOps One operations: consistent shell/layout primitives, clearer CMDB asset and relation workflows, improved discovery/notification operation feedback, and a repeatable frontend guardrail process aligned with manual-trigger CI policy.

## Highlights

- Completed v0.0.4 UI issue sequence (`#24` to `#27`) with production-facing UX consistency improvements.
- Added CMDB operator efficiency controls (search/filter/sort/read-only guidance/safer relation actions).
- Added automated and documented UI safety checks (`check:ui`) for release and iteration regression protection.

## Added

- Web-console reusable layout primitives:
  - `apps/web-console/src/components/layout.tsx`
  - `AuthGate`, `AppShell`, `SectionCard`
- LAN developer helper scripts:
  - `scripts/dev-lan-up.sh`
  - `scripts/dev-lan-down.sh`
- CMDB asset UX controls:
  - search input
  - status/class/site filters
  - sort modes
  - filtered empty-state reset action
- Discovery/notification operation feedback:
  - section summaries
  - loading-state messages
  - post-action confirmation banners
  - status chip visualization
- Frontend guardrail script and commands:
  - `apps/web-console/scripts/check-guardrails.mjs`
  - `npm run check:guardrails`
  - `npm run check:ui`
- UI verification documentation:
  - `docs/12-v0.0.4-ui-iteration-plan.md`
  - `docs/13-web-ui-quality-guardrails.md`

## Changed

- CI remains manual-trigger only (`workflow_dispatch`) and web quality step now executes `npm run check:ui`.
- Web-console default API endpoint resolution now follows current host (`<current-host>:8080`) when `VITE_API_BASE_URL` is not set, improving LAN usability.
- CMDB relations/discovery/notifications now provide explicit read-only guidance for non-writer roles instead of only hiding write actions.
- Web-console operation surfaces now follow a shared shell/card visual baseline for lower visual friction across sections.

## Fixed

- Prevented self-relation creation in CMDB relation flow (`source == target` validation).
- Added relation deletion confirmation to reduce accidental destructive actions.
- Improved visibility of action outcomes for discovery review and notification setup workflows via success notices.

## Breaking Changes

- None

## Upgrade Steps

1. Pull latest source from `main`.
2. No database migration is required specifically for `v0.0.4`; keep existing migration workflow unchanged.
3. Reinstall/update frontend dependencies and run UI checks:
   - `cd apps/web-console`
   - `npm ci`
   - `npm run check:ui`
4. For LAN usage, run helper scripts as needed:
   - `bash scripts/dev-lan-up.sh`
   - `bash scripts/dev-lan-down.sh`
5. Trigger GitHub CI manually when release-level verification is needed:
   - GitHub UI: `Actions -> CI -> Run workflow`
   - or CLI: `gh workflow run ci.yml`
