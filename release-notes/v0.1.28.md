## Executive Summary

v0.1.28 improves operator execution continuity from the overview cockpit. This release adds shortcut-first flow entry to setup/follow-up/escalation paths, introduces large-screen playlist run-state controls, and harmonizes key cockpit state cues to `ready/attention/blocking`.

## Highlights

- Operator flow shortcuts from overview to setup, follow-up, escalation, and next action.
- Demo control baseline for scene playlist run-state (`start/pause/next/reset`) with active scene visibility.
- Key cockpit block state-cue harmonization with one shared vocabulary: `ready`, `attention`, `blocking`.
- Release gate now enforces operator-flow acceptance artifact metadata for `v0.1.28+`.

## Upgrade Steps

1. Pull `v0.1.28` code and restart API/web services.
2. Open `#/overview` and verify:

   - scene playlist run-state controls,
   - unified state chips (`ready/attention/blocking`) in key cockpit blocks.
3. Validate release gates:
