# CloudOps One v0.1.4

## Release Metadata

- Version: `v0.1.4`
- Release date: `2026-03-06`
- Release type: `minor`
- Git tag: `v0.1.4`

## Executive Summary

v0.1.4 focuses on safer day-2 operations for teams without dedicated SRE staffing.

The release adds policy-gated high-risk playbook execution, two-person approval control, explainable alert suppression governance, no-code backup/DR operations, weekly digest export, and release-evidence tooling for long-soak and multi-region stability checks.

## Highlights

- Added policy and approval governance to high-risk playbook operations.
- Added no-code backup/DR workflow and weekly operations digest in cockpit.
- Added long-soak and multi-region simulation scripts to generate release-grade stability evidence.

## Changed

- Alert center remediation now requires explicit approval flow for high-risk playbook execution.
- Daily cockpit now includes backup/DR policy wizard, run history panel, and weekly digest generation/export controls.
- RBAC integration checks now include backup/DR and weekly digest API coverage.
- Release governance now supports synchronized publish/reconcile workflow via:
  - `make release-publish-dry`
  - `make release-publish`
  - `make release-check`

## Upgrade Steps

1. Pull `v0.1.4` code and apply migrations, including:
   - `202603060004_create_playbook_execution_policy.sql`
   - `202603060005_create_playbook_approval_requests.sql`
   - `202603060006_create_backup_dr_policy_tables.sql`
2. Restart API and web-console services.
3. Validate high-risk playbook policy and approval flow from `#/workflow`.
4. Validate backup/DR and weekly digest workflow from `#/overview` daily cockpit panel.
5. Generate long-soak and multi-region evidence artifacts before production rollout.
