# CloudOps One v0.1.16

## Release Metadata

- Version: `v0.1.16`
- Release date: `2026-03-15`
- Release type: `minor`
- Git tag: `v0.1.16`
- GitHub issues: `#199, #200, #201, #202, #203`

## Executive Summary

v0.1.16 shifts the operator experience from scattered setup inspection to one explicit go-live readiness workspace.

This release adds a cross-domain readiness snapshot, deterministic remediation metadata, cockpit inline guided actions, and a release-gate validation suite that proves read, apply, verify, render-contract, and read-only guard behavior.

## Highlights

- Added a single go-live readiness API across six core operating domains.
- Added deterministic remediation actions so operators can see the next step for each gap.
- Added a cockpit admin workspace that exposes overall status, next recommended domain, evidence, and inline actions.
- Added a v0.1.16 operator journey script that validates guided remediation and read-only enforcement.

## Added

- Planning/release docs:
  - `docs/52-v0.1.16-go-live-readiness-plan.md`
  - `docs/53-v0.1.16-release-gate-checklist.md`
- Go-live readiness endpoint:
  - `GET /api/v1/ops/cockpit/go-live/readiness`
- v0.1.16 validation suite:
  - `scripts/qa-v0.1.16-operator-journey.sh`

## Changed

- Go-live readiness domains now expose operator-facing metadata including:
  - `overall_status`,
  - `recommended_next_domain`,
  - per-domain `status`, `summary`, `reason`, `evidence`,
  - optional per-domain `recommended_action` with reusable API payloads or review links.
- Guided remediation now productizes two write-capable operator actions through the existing integration bootstrap API:
  - operator notification defaults,
  - default ticket follow-up routing.
- Cockpit runbook admin panel now adds:
  - go-live readiness workspace visibility,
  - inline guided action apply for auto-applicable domains,
  - immediate readiness refresh after related mutations.
- Roadmap governance now records v0.2.0 graduation criteria in:
  - `docs/20-v0.1x-operator-simplicity-plan.md`

## Fixed

- Operators no longer need to hunt across authentication, monitoring, notifications, backup, and runbook pages to understand whether the environment is ready for go-live.
- Guided remediation attempts are now consistently protected by RBAC while still remaining inspectable to read-only users.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.16` code and restart API/web services.
2. Ensure `v0.1.15` integration bootstrap APIs and related schema changes are already applied.
3. Verify the new API:
   - `GET /api/v1/ops/cockpit/go-live/readiness`
4. Open the cockpit runbook admin panel and verify the go-live readiness workspace appears above the integration bootstrap workspace.
5. Run v0.1.16 operator validation:
   - `bash scripts/qa-v0.1.16-operator-journey.sh --api-base-url http://127.0.0.1:8080 --auth-user admin`
   - `bash scripts/qa-v0.1.16-operator-journey.sh --api-base-url http://127.0.0.1:8080 --auth-user viewer --read-only`

## Known Issues

- v0.1.16 focuses on scan-first go-live readiness and does not yet convert every setup domain into a full inline configuration workflow.
- Workspace validation remains API-first and does not include browser automation.
