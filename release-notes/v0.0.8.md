# CloudOps One v0.0.8

## Release Metadata

- Version: `v0.0.8`
- Release date: `2026-03-05`
- Release type: `minor`
- Git tag: `v0.0.8`

## Executive Summary

v0.0.8 delivers the discovery scheduler and ticket domain baseline, hardens monitoring secret handling,
adds runtime i18n language switching, and introduces a stronger performance/reliability toolkit with
index-backed read-path optimization and benchmark stress enhancements.

## Highlights

- Added scheduler-driven CMDB discovery execution with runtime controls.
- Added ticket APIs and web-console ticket dashboard/detail workflows with RBAC coverage.
- Added monitoring secret encryption-at-rest and masked secret outputs.
- Added runtime language switching (`en-US`, `zh-CN`) with locale coverage checks.
- Added read-path/perf hardening: monitoring-layer SQL aggregation, class-filter normalization, and new DB indexes.
- Added benchmark upgrades: API parallel profile (`--concurrency`) and SSE stress profile (`--burst-count`) with lag distribution.

## Added

- Discovery scheduler baseline:
  - migration: `202603040002_add_discovery_job_next_run_at.sql`
  - worker: `services/api/src/discovery_scheduler_worker.rs`
  - env controls: `DISCOVERY_SCHEDULER_ENABLED`, `DISCOVERY_SCHEDULER_POLL_SECONDS`
- Ticket domain baseline:
  - migrations: `202603040003_create_ticket_tables.sql`, `202603040004_add_ticket_permissions.sql`
  - API module: `services/api/src/tickets.rs`
  - web-console ticket views and route wiring
- Monitoring source secret hardening:
  - migration: `202603040005_encrypt_monitoring_source_secrets.sql`
  - shared secret module: `services/api/src/secrets.rs`
  - masked secret outputs for API/probe paths
- Performance index baseline:
  - migrations: `202603050001_optimize_assets_and_monitoring_indexes.sql`, `202603050002_add_cmdb_asset_filter_indexes.sql`
  - new indexes for monitoring-layer scans, latest sync-job lookup, and CMDB filtered list paths

## Changed

- Workflow execution policy now enforces explicit modes (`disabled`, `allowlist`, `sandboxed`).
- Web console supports runtime language switching with persisted locale preference.
- Monitoring layer endpoint now computes summary counters in SQL and reuses summary `asset_total` as list total.
- Monitoring layer filtering now uses explicit normalized class sets instead of CASE classification SQL.
- Audit best-effort writes now run via detached async tasks and emit slow-write warnings (`>500ms`).
- Monitoring sync enqueue path now resolves asset existence/class in one query.
- API benchmark script now supports `--concurrency` and stage-level utilization snapshots.
- SSE benchmark script now supports `--burst-count` and lag distribution reporting (`min/avg/p50/p95/p99/max`).

## Fixed

- Unauthorized ticket list route handling now aligns with RBAC mapping.
- Web guardrail checks are now aligned with modular page architecture after App refactor.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull release code and install dependencies for `v0.0.8`.
2. Run database migrations, including:
   - `202603040002_add_discovery_job_next_run_at.sql`
   - `202603040003_create_ticket_tables.sql`
   - `202603040004_add_ticket_permissions.sql`
   - `202603040005_encrypt_monitoring_source_secrets.sql`
   - `202603050001_optimize_assets_and_monitoring_indexes.sql`
   - `202603050002_add_cmdb_asset_filter_indexes.sql`
3. Restart API and worker processes.
4. Re-run web quality gate and API checks:
   - `cd apps/web-console && npm run check:ui`
   - `cargo check --workspace`
   - `cargo test -p api`
5. Re-run benchmark baselines as needed:
   - `bash scripts/benchmark-api-load.sh --concurrency 10`
   - `bash scripts/benchmark-sse-burst-smoke.sh --burst-count 300`
