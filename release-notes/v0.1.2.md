# CloudOps One v0.1.2

## Release Metadata

- Version: `v0.1.2`
- Release date: `2026-03-05`
- Release type: `minor`
- Git tag: `v0.1.2`

## Executive Summary

v0.1.2 hardens enterprise authentication paths and strengthens scale-readiness validation for small and medium operations teams. The release adds LDAP/local auth guardrails, local MFA/session/lockout controls, and profile-based benchmark quality gates for 1k to 5k asset scenarios.

## Highlights

- Delivered enterprise identity hardening for LDAP and local fallback accounts.
- Added local password, MFA enrollment, and local login APIs with policy and audit controls.
- Added profile-based benchmark orchestration, profile-aware SLO gate, and trend-delta reporting for release evidence.

## Added

- LDAP login baseline API:
  - `POST /api/v1/auth/ldap/login`
- Local auth hardening APIs:
  - `POST /api/v1/auth/local/password`
  - `POST /api/v1/auth/local/mfa/enroll`
  - `POST /api/v1/auth/local/login`
- Benchmarking tooling:
  - `scripts/benchmark-scale-profiles.sh`
  - `scripts/benchmark-trend-delta.sh`
- Local auth lockout validation script:
  - `scripts/test-local-auth-hardening.sh`
- Security + scale closure evidence doc:
  - `docs/23-v0.1.2-security-scale-evidence.md`

## Changed

- Benchmark scripts now support `smoke`, `scale-1k`, and `scale-5k` profile presets and emit profile metadata artifacts.
- Threshold gate policy is profile-aware and fails fast on unknown profile configuration.
- Local fallback policy supports explicit governance modes:
  - `allow_all`
  - `break_glass_only`
  - `disabled`
- Local auth policy now supports configurable session and brute-force controls:
  - `AUTH_LOCAL_SESSION_IDLE_TIMEOUT_MINUTES`
  - `AUTH_LOCAL_SESSION_MAX_AGE_MINUTES`
  - `AUTH_LOCAL_SESSION_MAX_CONCURRENT`
  - `AUTH_LOCAL_LOCKOUT_THRESHOLD`
  - `AUTH_LOCAL_LOCKOUT_MINUTES`
  - `AUTH_LOCAL_RATE_LIMIT_WINDOW_SECONDS`
  - `AUTH_LOCAL_RATE_LIMIT_MAX_ATTEMPTS`
- Performance baseline documentation now includes tuned profile SLO thresholds for v0.1.2.

## Fixed

- Added deterministic local-session idle-timeout handling in bearer-token validation path.
- Added lockout counter clear-on-success path after successful local authentication.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.2` code and apply database migrations.
2. Restart API and web-console services.
3. Configure local auth policy env vars based on enterprise security baseline.
4. Verify LDAP login and local break-glass flows in a staging environment.
5. Run profile benchmark gate and trend-delta commands before production rollout.
6. Validate results using commands listed in the Validation section.

## Known Issues

- LDAP `live` mode is not implemented yet; current release includes dev-mode baseline plus mapping policy enforcement.
- Local password hashing uses SHA-256 with salt and should be upgraded to Argon2id/scrypt in a follow-up release.
- SSE trend delta currently shows partial missing baseline metrics when older summary files do not include lag distributions.
- Scale evidence is based on a local single-node environment; multi-node soak validation remains pending.
