# CloudOps One v0.1.5

## Release Metadata

- Version: `v0.1.5`
- Release date: `2026-03-07`
- Release type: `minor`
- Git tag: `v0.1.5`

## Executive Summary

v0.1.5 strengthens no-code operations continuity by adding change-window visibility, shift handover carryover controls, restore verification evidence tracking, and release-grade simulation and validation tooling.

The focus is reducing cross-shift blind spots and surfacing operational conflicts before high-risk changes are executed.

## Highlights

- Added unified change calendar overlays and conflict checks in cockpit.
- Added shift handover digest with required carryover ownership/action closure fields.
- Added restore evidence workflow for backup and drill confidence tracking.
- Added reproducible escalation/failover simulation and one-command operator journey validation tooling.

## Added

- Change calendar APIs:
  - `GET /api/v1/ops/cockpit/change-calendar`
  - `POST /api/v1/ops/cockpit/change-calendar/conflicts`
- Restore evidence schema and APIs:
  - migration: `services/api/migrations/202603070003_create_restore_verification_evidence.sql`
  - `GET /api/v1/ops/cockpit/backup/restore-evidence`
  - `POST /api/v1/ops/cockpit/backup/runs/{id}/restore-evidence`
  - `PATCH /api/v1/ops/cockpit/backup/restore-evidence/{id}`
- Handover digest schema and APIs:
  - migration: `services/api/migrations/202603070004_create_handover_digest_tables.sql`
  - `GET /api/v1/ops/cockpit/handover-digest`
  - `GET /api/v1/ops/cockpit/handover-digest/export?format=csv|json`
  - `POST /api/v1/ops/cockpit/handover-digest/items/{item_key}/close`
- Simulation tooling:
  - `scripts/ops-escalation-failover-sim.sh`
  - `scripts/ops-escalation-failover-policy.json`
  - `docs/30-v0.1.5-escalation-failover-simulation.md`
- Validation tooling:
  - `scripts/qa-v0.1.5-operator-journey.sh`
  - `docs/31-v0.1.5-operator-journey-validation.md`

## Changed

- High-risk playbook execution policy responses now include calendar conflict details and recommended next slot context.
- Daily cockpit UI now includes:
  - change calendar range view with conflict checker,
  - restore evidence attach/close workflow,
  - shift handover digest generation/export/close actions.
- Handover digest export uses deterministic timestamp derivation for reproducible export artifacts.
- RBAC coverage now includes change-calendar, handover-digest, and restore-evidence route matrix checks.

## Fixed

- `scripts/test-rbac-policy.sh` now initializes ticket context before handover close checks, avoiding invalid variable ordering during integration assertions.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.5` code and apply migrations:
   - `202603070003_create_restore_verification_evidence.sql`
   - `202603070004_create_handover_digest_tables.sql`
2. Restart API and web-console services.
3. Validate cockpit flows in `#/overview`:
   - change calendar overlays/conflict check,
   - restore evidence attach/close,
   - handover digest generation/export/close.
4. Run simulation and validation evidence commands:
   - `bash scripts/ops-escalation-failover-sim.sh --allow-fail`
   - `bash scripts/qa-v0.1.5-operator-journey.sh --read-only` (or full write-path mode in staging)
