## Executive Summary

v0.1.31 is a UX-only release focused on reducing first-screen overload and making navigation predictable for new and non-technical operators. The console now defaults to a focused page mode with essential content, while advanced content remains available on demand.

## Highlights

- Added deterministic Focus Mode and Advanced Mode toggle across major pages.
- Stabilized navigation hierarchy as grouped menu (L1), in-group tabs (L2), and section tabs (L3).
- Added UX acceptance artifacts as a mandatory release-gate contract for `v0.1.31+`.

## Added

- v0.1.31 planning and gate docs:
  - `docs/84-v0.1.31-usability-ux-only-plan.md`
  - `docs/85-v0.1.31-release-gate-checklist.md`
- Localized section-tab and page-mode labels (`en-US`, `zh-CN`).
- UX acceptance evidence set:
  - `.run/ux-acceptance/v0.1.31/*`

## Changed

- Page rendering now defaults to essential first-screen sections in Focus Mode.
- Advanced sections are shown only after explicit user toggle.
- Release governance tooling now validates `UX acceptance artifacts` metadata and file existence for `>= v0.1.31`.

## Fixed

- Reduced mixed-content clutter that made first actions unclear on entry.
- Improved section navigation clarity with deterministic active section-tab state.
- Prevented release omissions for UX evidence by extending gate enforcement.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.31` and restart API/web services.
2. Open web console and verify default Focus Mode on key pages (`overview`, `cmdb`, `monitoring`, `workflow`).
3. Toggle `Show Advanced` and verify advanced sections become available.
4. Verify release governance gates including UX acceptance metadata:
   - `bash scripts/release-github-issues-check.sh --version 0.1.31 --notes-file release-notes/v0.1.31.md`

## Known Issues

- Focus-mode essential section selection is currently static by page and will be made user-configurable in subsequent versions.
- Advanced mode state resets on page switch by design in this release; user preference persistence is not yet implemented.
