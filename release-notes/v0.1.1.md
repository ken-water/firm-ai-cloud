# CloudOps One v0.1.1

## Release Metadata

- Version: `v0.1.1`
- Release date: `2026-03-05`
- Release type: `minor`
- Git tag: `v0.1.1`

## Executive Summary

v0.1.1 extends the operator-simplicity baseline with guided day-2 operations:

- safe playbook library with dry-run and explicit confirmation gates,
- a unified daily operations cockpit queue with deterministic priorities,
- topology edge diagnostics with trend/alerts/change context and guided checklist actions.

The release is designed for small and medium enterprise teams that need reliable operations without deep scripting expertise.

## Highlights

- Added first-class playbook catalog and safe parameter schema model with built-in operations playbooks.
- Enforced high-risk playbook execution policy (`dry-run` plus explicit confirmation token).
- Added daily cockpit aggregation API and UI with rationale-rich prioritized queue and next actions.
- Added topology edge diagnostics APIs and UI panel for guided troubleshooting context.

## Added

- Playbook catalog + execution persistence:
  - migration: `services/api/migrations/202603050004_create_playbook_catalog_and_execution_logs.sql`
  - built-in playbooks: `restart-service-safe`, `drain-node-maintenance`, `refresh-monitoring-binding`, `rotate-app-secret`, `collect-topology-diagnostics`
- Playbook APIs:
  - `GET /api/v1/workflow/playbooks`
  - `GET /api/v1/workflow/playbooks/{key}`
  - `POST /api/v1/workflow/playbooks/{key}/dry-run`
  - `POST /api/v1/workflow/playbooks/{key}/execute`
  - `GET /api/v1/workflow/playbooks/executions`
  - `GET /api/v1/workflow/playbooks/executions/{id}`
  - `POST /api/v1/workflow/playbooks/executions/{id}/replay`
- Daily cockpit aggregation API:
  - `GET /api/v1/ops/cockpit/queue`
- Topology diagnostics API:
  - `GET /api/v1/topology/diagnostics/edges/{edge_id}`
- Web console enhancements:
  - workflow page playbook library section (`#/workflow`)
  - daily operations cockpit queue section (`#/overview`, `#/monitoring`)
  - topology diagnostics panel and quick actions (`#/topology`)
- Validation/runbook artifacts:
  - `scripts/qa-v0.1.1.sh`
  - `docs/21-v0.1.1-validation-suite.md`
  - `docs/22-v0.1.1-operator-runbook.md`

## Changed

- RBAC route matrix now includes:
  - `workflow.playbooks.read`
  - `workflow.playbooks.write`
  - `ops.cockpit.read`
- RBAC integration script now validates playbook and cockpit permissions for operator/viewer users.
- Developer quickstart now includes v0.1.1 API smoke commands.

## Fixed

- Added deterministic multi-source queue ordering to avoid unstable cockpit item ordering across repeated refreshes.
- Added strict topology diagnostics window validation to prevent invalid query ranges.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.1` code and apply database migrations.
2. Restart API and web-console services.
3. Verify playbook catalog and dry-run flow from `#/workflow`.
4. Verify daily cockpit queue from `#/overview` and one next-action execution.
5. Verify topology edge diagnostics panel from `#/topology`.
6. Run validation commands listed in the Validation section.
