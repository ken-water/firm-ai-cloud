## Executive Summary

v0.1.32 focuses on user adoption friction instead of feature expansion. This release restructures navigation around user tasks, adds a consistent page-start entry pattern, and upgrades visual hierarchy so users can decide what to do faster on first entry.

## Highlights

- Task-first navigation grouping and user-language menu labels.
- Unified page-start panel with first-action and quick section entry.
- Non-disruptive modern visual uplift with stronger hierarchy and readability.
- New release gate enforcement for measurable usability acceptance artifacts.

## Changed

- Navigation groups and labels now prioritize user tasks over internal module naming.
- Core pages now expose a start panel with page purpose and direct first action.
- Visual hierarchy was updated for better first-screen readability while preserving existing task paths.
- Release governance now enforces `Usability acceptance artifacts` for `>= v0.1.32`.

## Fixed

- Reduced entry ambiguity in menu and page-level task discovery.
- Reduced first-screen overload by exposing deterministic action entry first.
- Reduced governance drift by failing releases that miss usability evidence metadata.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.32` and restart API/web services.
2. Open web console and verify the new grouped menu and page-start panel behavior.
3. Validate release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`
   - `bash scripts/release-github-issues-check.sh --version 0.1.32 --notes-file release-notes/v0.1.32.md`
