# CloudOps One v0.1.9

## Release Metadata

- Version: `v0.1.9`
- Release date: `2026-03-09`
- Release type: `minor`
- Git tag: `v0.1.9`
- GitHub issues: `#157, #158, #159, #160, #161`

## Executive Summary

v0.1.9 introduces runbook analytics visibility for operators who need fast, low-risk triage decisions without deep technical context.

This release adds summary metrics and a failure-focused feed at API and cockpit UI layers, then closes the release gate with an end-to-end operator validation suite.

## Highlights

- Added runbook analytics summary API with bounded time-window filtering.
- Added runbook failure feed API for remediation prioritization.
- Added cockpit analytics panel and failure hotspot table in runbook workspace.
- Added v0.1.9 operator journey validation script and release-gate checklist.

## Changed

- Cockpit runbook panel now supports:
  - analytics filter controls (`days`, `execution_mode`, template scope),
  - summary view (totals, success rate, replay usage, failed-step hotspots),
  - failure hotspot feed with diagnostics and remediation hints.
- Daily cockpit snapshot loading now includes runbook analytics summary and failure feed data pulls.
- RBAC route coverage tests now include runbook analytics endpoints.

## Upgrade Steps

1. Pull `v0.1.9` code and restart API/web services.
2. Verify new analytics endpoints:
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/summary`
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/failures`
3. Open cockpit runbook panel and validate analytics summary/failure hotspot sections.
4. Run v0.1.9 operator validation:
   - `bash scripts/qa-v0.1.9-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.9-operator-journey.sh --auth-user viewer --read-only`

## Known Issues

- Analytics summary scans are intentionally bounded; very large time windows can be marked as truncated.
