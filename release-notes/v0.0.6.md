# CloudOps One v0.0.6

## Release Metadata

- Version: `v0.0.6`
- Release date: `2026-03-03`
- Release type: `minor`
- Git tag: `v0.0.6`

## Executive Summary

v0.0.6 strengthens CMDB governance and onboarding safety by adding lifecycle readiness gates,
discovery candidate decision hardening, asynchronous CMDB-to-monitoring provisioning, and
hierarchy-aware impact traversal with readiness-focused console workflows.

## Highlights

- Enforced lifecycle progression and operational readiness checks (`idle -> onboarding -> operational -> maintenance -> retired`).
- Hardened discovery onboarding by requiring explicit candidate review strategies before CMDB acceptance.
- Added async CMDB-to-monitoring sync pipeline for Zabbix onboarding with binding and job visibility.
- Improved hierarchy/dependency modeling for physical host, VM, container, and business-service chains.
- Delivered readiness and multi-binding management UX in web console.

## Added

- Asset lifecycle and binding schema + APIs:
  - `202603030002_create_asset_bindings_and_lifecycle.sql`
  - `GET/PUT /api/v1/cmdb/assets/{id}/bindings`
  - `POST /api/v1/cmdb/assets/{id}/lifecycle`
- Discovery candidate review hardening migration and metadata:
  - `202603030003_harden_discovery_candidate_review.sql`
  - `review_strategy`, `review_reason`, `review_asset_id`
- CMDB monitoring sync baseline:
  - `202603030004_create_cmdb_monitoring_sync.sql`
  - `cmdb_monitoring_sync_jobs`, `cmdb_monitoring_bindings`
  - `GET /api/v1/cmdb/assets/{id}/monitoring-binding`
  - `POST /api/v1/cmdb/assets/{id}/monitoring-sync`
  - `GET /api/v1/cmdb/monitoring-sync/jobs`
- Hierarchy and impact traversal baseline:
  - `202603030005_standardize_relation_types_and_hierarchy_indexes.sql`
  - `GET /api/v1/cmdb/assets/{id}/impact`

## Changed

- Candidate approval flow now supports deterministic decision controls (`auto`, `create`, `merge`) and explicit target selection.
- Discovery dedup behavior now refreshes existing pending candidates for matching identity signals.
- Relation API now canonicalizes common aliases to standardized relation types.
- Web console now includes readiness/binding management and lifecycle transition controls.

## Fixed

- Prevented duplicate pending candidates during repeated discovery runs.
- Prevented invalid `contains` hierarchy cycles and conflicting multi-parent child edges.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull latest code for `v0.0.6` and update dependencies.
2. Run database migrations (includes `202603030002` to `202603030005`).
3. Restart API service to activate monitoring sync worker and new CMDB routes.
4. Verify RBAC mappings for CMDB monitoring sync endpoints.

## Validation

- Backend: `cargo test -p api` passed.
- Frontend: `npm --prefix apps/web-console run check:ui` passed.
- Smoke tests: `bash scripts/test-cmdb-loop.sh` passed for discovery -> approve(create/merge) flow.

## Known Issues

- Existing deployments still need environment-specific Zabbix template/proxy mapping tuning for production.
