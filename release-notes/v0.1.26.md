## Executive Summary

v0.1.26 improves SMB decision speed by adding role-focused cockpit quick scenes, deterministic AI follow-up intent cards, and a topology-owner follow-up lens. Release governance is also hardened with role-cockpit acceptance artifact checks for `v0.1.26+`.

## Highlights

- Added one-click role cockpit scene switch (`operator/admin/network/business`).
- Added AI follow-up intent cards with deterministic `intent/evidence/action/guard/status` mapping.
- Added topology ownership follow-up lens with deterministic priority ranking.
- Release gate now enforces role-cockpit acceptance artifact metadata.

## Added

- `v0.1.26` planning and release-gate docs:
  - `docs/74-v0.1.26-role-cockpit-ai-followup-plan.md`
  - `docs/75-v0.1.26-release-gate-checklist.md`
- Cockpit role scene metadata and quick switch flow.
- Cockpit topology ownership follow-up lens panel.

## Changed

- Dashboard layout reset now resets active scene to `operator` to keep behavior deterministic.
- AI evidence area now exposes follow-up intent cards before guided-action table for faster triage.
- Release issues gate now validates role-cockpit acceptance artifact paths for `v0.1.26+`.

## Fixed

- Reduced cockpit navigation friction for role-focused demos and daily use.
- Reduced AI evidence interpretation overhead by adding card-level follow-up summaries.
- Reduced topology-to-owner follow-up context switching.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.26` code and restart API/web services.
2. Open `#/overview` and verify role scene switch bar is visible.
3. Run AI evidence query in overview and verify follow-up intent cards are rendered.
4. Verify topology ownership follow-up lens appears in business topology section.
5. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`

## Known Issues

- Role scenes are local layout presets and do not yet include server-side role profile synchronization.
- AI follow-up intent cards are read-model summaries and do not execute actions directly.
