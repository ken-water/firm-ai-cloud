- Version: `v0.1.10`

## Executive Summary

v0.1.10 closes the runbook risk-governance gap by adding configurable analytics policy controls and proactive risk alerts, so operators can identify unstable templates before repeated failures escalate.

This release delivers API, cockpit UX, and validation-script coverage for policy read/write, alert visibility, and runbook execution record checks.

## Highlights

- Added runbook analytics policy persistence and read/write APIs.
- Added proactive runbook risk alert feed API with severity and remediation guidance.
- Added cockpit risk policy editor and risk alerts panel in runbook workspace.
- Added v0.1.10 operator journey validation script with required stage keys and artifact outputs.

## Upgrade Steps

1. Pull `v0.1.10` code and restart API/web services.
2. Apply migration `202603100001_create_runbook_analytics_policies.sql`.
3. Verify new endpoints:
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/policy`
   - `PUT /api/v1/ops/cockpit/runbook-templates/analytics/policy`
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/alerts`
4. Open cockpit runbook panel and validate risk policy and risk alerts sections.
5. Run v0.1.10 operator validation:
   - `bash scripts/qa-v0.1.10-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.10-operator-journey.sh --auth-user viewer --read-only`
