# CloudOps One v0.1.21

## Release Metadata

- Version: `v0.1.21`
- Release date: `2026-03-18`
- Release type: `minor`
- Git tag: `v0.1.21`
- GitHub issues: `#229, #230, #231, #232, #233`

## Executive Summary

v0.1.21 establishes the module-first dashboard baseline and lands the first business/org/AI contract set required by the product strategy baseline. Operators now get one cockpit entry to view module risk, business service risk/resource context, cross-department workflow routing baseline, and evidence-backed AI query results.

## Highlights

- Module-first operation cockpit is now the explicit overview entry point with configurable widget layout persistence.
- Business-centric monitoring/CMDB baseline is now queryable and visible in cockpit.
- Organization workflow baseline now exposes cross-department approval/routing semantics and guardrails.
- AI entry now returns evidence-backed answers with required source/time metadata.

## Added

- Planning and release gate docs:
  - `docs/63-v0.1.21-dashboard-business-org-ai-plan.md`
  - `docs/64-v0.1.21-release-gate-checklist.md`
- Cockpit APIs:
  - `GET /api/v1/ops/cockpit/business-overview`
  - `GET /api/v1/ops/cockpit/workflow-org-baseline`
  - `POST /api/v1/ops/cockpit/ai/evidence-query`

## Changed

- Overview `section-module-cockpit` now includes:
  - business risk/resource baseline table,
  - organization workflow baseline panel,
  - AI evidence query entry and evidence table.
- Dashboard widget layout contract remains browser-local and deterministic (`cloudops.dashboard_layout.v1`), with refresh/reset controls.

## Fixed

- Business/service, workflow/org, and AI/evidence strategy requirements are now represented by product contracts and not left as planning-only items.
- AI answer payloads now enforce evidence completeness guardrails before returning responses.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.21` code and restart API/web services.
2. Ensure API and web-console are running with latest migrations.
3. Verify cockpit baselines:
   - `GET /api/v1/ops/cockpit/business-overview`
   - `GET /api/v1/ops/cockpit/workflow-org-baseline`
   - `POST /api/v1/ops/cockpit/ai/evidence-query`
4. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`

## Validation

- Backend:
  - `cargo check -p api` passed.
  - `cargo test -p api` passed.
- Frontend:
  - `cd apps/web-console && npm run check:ui` passed.

## Known Issues

- Workflow parallel approval execution remains baseline-contract level in this release and is currently represented as serial emulation in practice.
- AI module currently supports baseline summary intents for monitoring/CMDB/workflow and does not execute write actions.
