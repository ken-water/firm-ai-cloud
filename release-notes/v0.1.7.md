# CloudOps One v0.1.7

## Release Metadata

- Version: `v0.1.7`
- Release date: `2026-03-09`
- Release type: `minor`
- Git tag: `v0.1.7`
- GitHub issues: `#145, #146, #147, #148, #149, #150`

## Executive Summary

v0.1.7 closes the runbook execution policy gap from v0.1.6 by introducing explicit execution governance and a guarded hybrid-live execution path.

This release keeps non-technical operator UX simple while reducing ambiguity between simulation and live checks.

## Highlights

- Added runbook execution policy APIs with audit coverage.
- Added `execution_mode` support (`simulate|live`) for runbook execution.
- Added guarded live adapter for `dependency-check` (bounded TCP probe).
- Added runtime summary fields in runbook execution records.
- Added v0.1.7 operator validation suite and release gate checklist.

## Changed

- Runbook template catalog now includes `execution_modes` for each template.
- `dependency-check` template supports hybrid live mode under policy allowlist.
- Web console runbook panel now includes:
  - execution policy read/write form,
  - mode selector for template execution,
  - runtime summary rendering in execution history rows.
- Product planning index now includes v0.1.7 docs and queue.

## Upgrade Steps

1. Pull `v0.1.7` code and apply migrations:
   - `202603090001_create_runbook_execution_policy.sql`
2. Restart API and web-console services.
3. Verify runbook policy endpoint:
   - `GET /api/v1/ops/cockpit/runbook-templates/execution-policy`
4. Validate runbook execution modes in cockpit:
   - template catalog exposes `execution_modes`
   - simulate and live execution paths behave per policy
5. Run v0.1.7 operator validation:
   - `bash scripts/qa-v0.1.7-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.7-operator-journey.sh --auth-user viewer --read-only`

## Known Issues

- Hybrid live adapter currently supports only `dependency-check`; other templates remain simulation-only.
- Live probe validates TCP reachability and does not perform full business-transaction verification.
- Full write-path validation still requires a reachable API environment with suitable role bindings.
