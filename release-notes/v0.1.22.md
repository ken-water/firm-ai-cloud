# CloudOps One v0.1.22

## Release Metadata

- Version: `v0.1.22`
- Release date: `2026-03-18`
- Release type: `minor`
- Git tag: `v0.1.22`
- GitHub issues: `#235, #236, #237, #238, #239`

## Executive Summary

v0.1.22 completes the first usability hardening round after v0.1.21 baseline contracts. This release ships dashboard studio role templates, business topology risk visibility, AI copilot intent presets with evidence continuity, and one-screen operator handoff clarity.

## Highlights

- Dashboard studio v1 now supports deterministic role-template composition.
- Business topology risk is now visible at service level with cross-site and risk signals.
- AI copilot now provides preset-driven query intents and evidence continuity hints.
- Operators now have one-screen handoff focus list with deterministic priority ordering.

## Changed

- Module cockpit now includes:
  - dashboard role template panel (operator/admin/network/business),
  - business topology risk panel,
  - AI preset-driven query flow,
  - one-screen operator handoff panel.
- AI evidence panel now surfaces continuity-friendly source references.

## Fixed

- Dashboard customization now has a reproducible baseline path for non-technical SMB operators.
- Cross-domain handoff focus is now readable in one panel instead of fragmented across multiple sections.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.22` code and restart API/web services.
2. Ensure API and web-console are running with latest migrations.
3. Verify new endpoints:
   - `GET /api/v1/ops/cockpit/business-topology-overview`
   - `GET /api/v1/ops/cockpit/ai/intent-presets`
4. Run release gates:
   - `cargo check --workspace`
   - `cargo test -p api`
   - `cd apps/web-console && npm run check:ui`

## Known Issues

- AI preset list is baseline-driven and currently static; tenant/user personalization is planned in a later cycle.
- One-screen handoff action rows are read-model hints and not yet fully action-orchestration backed.
