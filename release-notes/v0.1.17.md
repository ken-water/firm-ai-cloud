# CloudOps One v0.1.17

## Release Metadata

- Version: `v0.1.17`
- Release date: `2026-03-16`
- Release type: `minor`
- Git tag: `v0.1.17`
- GitHub issues: `#205, #206, #207, #208, #209`

## Executive Summary

v0.1.17 shifts the setup experience toward first-value activation so SMB pilots can reach visible operational value faster and feed real friction back into the product.

This release adds an activation status API, opinionated SMB starter-template guidance, structured pilot-feedback capture, a setup activation workspace, and a release-gate validation suite that proves read, apply, verify, feedback-capture, and read-only guard behavior.

## Highlights

- Added a product-level first-value activation API for setup and pilot workflows.
- Added SMB starter-template guidance so new users can choose an opinionated baseline instead of starting from zero.
- Added structured activation-feedback capture to turn pilot friction into product-owned data.
- Added a setup activation workspace and a v0.1.17 operator validation script.

## Changed

- Setup now exposes operator-facing activation metadata including:
  - `overall_status`,
  - `recommended_next_step_key`,
  - `recommended_profile_key`,
  - activation items with `status`, `summary`, `reason`, `recommended_action`, and `evidence`.
- Setup page now adds:
  - first-value activation workspace visibility,
  - starter-template recommendations,
  - deterministic next-step links,
  - inline pilot-feedback submission.
- Setup template/profile apply and revert flows now refresh activation state so operators can see progress toward first value immediately after each change.
- Roadmap governance now explicitly ties short-term pilot learning to long-term SMB platform direction in:
  - `docs/54-v0.1.17-first-value-activation-plan.md`
  - `docs/20-v0.1x-operator-simplicity-plan.md`

## Fixed

- SMB pilots no longer need to guess which setup action should come next after installation and first login.
- Product teams can now capture pilot friction in structured activation events instead of relying on unstructured external notes.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.17` code and restart API/web services.
2. Ensure the new activation feedback migration is applied.
3. Verify the new APIs:
   - `GET /api/v1/setup/activation`
   - `GET /api/v1/setup/activation/starter-templates`
   - `POST /api/v1/setup/activation/feedback`
4. Open the setup page and verify the first-value activation workspace appears above the step-by-step setup controls.
5. Run v0.1.17 operator validation:
   - `bash scripts/qa-v0.1.17-activation-journey.sh --api-base-url http://127.0.0.1:8080 --auth-user admin`
   - `bash scripts/qa-v0.1.17-activation-journey.sh --api-base-url http://127.0.0.1:8080 --auth-user viewer --read-only`

## Known Issues

- v0.1.17 improves first-value activation but does not yet convert every starter-template recommendation into a one-click full setup flow.
- Activation feedback is currently product-owned data capture and does not yet include built-in analytics dashboards or trend reporting.
