# CloudOps One v0.1.6

## Release Metadata

- Version: `v0.1.6`
- Release date: `2026-03-08`
- Release type: `minor`
- Git tag: `v0.1.6`

## Executive Summary

v0.1.6 focuses on operator simplicity v2: guided setup presets, deterministic next actions, conflict-aware change reservation, continuity quality gates, and no-code runbook templates.

This release reduces decision overhead for non-specialist operators while improving handover and restore-evidence closure discipline.

## Highlights

- Added setup profile presets with preview/apply/revert path for no-code onboarding.
- Added cockpit next-best-action assistant and reservation-aware scheduling workflow.
- Added handover overdue reminders and ownership quality signals.
- Added restore evidence SLA policy + weekly compliance scorecard with export.
- Added one-click runbook templates with preflight checklist, timeline, evidence, and remediation hints.
- Added v0.1.6 one-command validation suite with per-stage diagnostics and artifact index.

## Upgrade Steps

1. Pull `v0.1.6` code and apply migrations:
   - `202603080001_create_setup_operator_profile_runs.sql`
   - `202603080002_create_change_calendar_reservations.sql`
   - `202603080003_create_restore_evidence_compliance_policy.sql`
   - `202603080004_create_ops_runbook_template_executions.sql`
2. Restart API and web-console services.
3. Validate cockpit paths in `#/overview`:
   - next actions and reservation workflow,
   - handover reminder/export,
   - evidence compliance policy/scorecard,
   - runbook templates execution timeline.
4. Run v0.1.6 operator validation:
   - `bash scripts/qa-v0.1.6-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.6-operator-journey.sh --auth-user viewer --read-only`
