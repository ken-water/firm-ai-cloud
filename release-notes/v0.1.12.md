# CloudOps One v0.1.12

## Release Metadata

- Version: `v0.1.12`
- Release date: `2026-03-11`
- Release type: `minor`
- Git tag: `v0.1.12`
- GitHub issues: `#176, #180, #179, #178, #175`

## Executive Summary

v0.1.12 closes the next operator follow-up gap after risk-to-ticket linkage by making risk alerts accountable end to end: each ticket action now records who owns the follow-up and what notification evidence was produced.

This release delivers backend notification evidence, deterministic owner routing, cockpit owner/dispatch visibility, and a validation script that covers create, route, notify, verify, and read-only guard paths.

## Highlights

- Added notification dispatch evidence and read model for runbook-risk ticket actions.
- Added deterministic owner routing with persisted ticket assignee and route metadata.
- Added cockpit visibility for owner route and dispatch outcome per risk alert row.
- Added v0.1.12 operator validation script with route and notification verification stages.

## Changed

- `POST /api/v1/ops/cockpit/runbook-templates/analytics/alerts/tickets` now:
  - dispatches `runbook_risk.ticket_linked` notifications,
  - persists delivery evidence rows,
  - returns `notification_summary`.
- Runbook risk alert payload now exposes:
  - `ticket_link.ticket_assignee`,
  - `ticket_link.owner_route`,
  - `notification_summary`.
- Cockpit runbook risk alert panel now supports:
  - owner-route and assignee rendering,
  - dispatch summary rendering,
  - notice feedback for created/reused ticket owner and notify state.
- v0.1.12 scope, dependency order, and release gate remain documented in:
  - `docs/20-v0.1x-operator-simplicity-plan.md`
  - `docs/44-v0.1.12-runbook-risk-dispatch-routing-plan.md`
  - `docs/45-v0.1.12-release-gate-checklist.md`

## Fixed

- Operators no longer need to inspect raw ticket metadata to determine who owns runbook-risk follow-up.
- Operators can now verify when no subscription matched because `skipped` delivery evidence is persisted explicitly.

## Breaking Changes

- None.

## Upgrade Steps

1. Pull `v0.1.12` code and restart API/web services.
2. Apply migration `202603110001_create_runbook_risk_alert_notification_deliveries.sql`.
3. Verify new/updated APIs:
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/alerts`
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/alerts/notifications`
   - `POST /api/v1/ops/cockpit/runbook-templates/analytics/alerts/tickets`
4. Open cockpit runbook panel and validate owner route plus dispatch summary visibility.
5. Run v0.1.12 operator validation:
   - `bash scripts/qa-v0.1.12-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.12-operator-journey.sh --auth-user viewer --read-only`
