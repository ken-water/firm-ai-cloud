## Release Metadata

## Executive Summary

v0.1.11 closes the runbook risk follow-up gap by turning proactive risk alerts into accountable ticket workflows directly in cockpit, with deterministic create/reuse behavior and visible linkage status.

This release delivers backend linkage persistence, one-click action API, cockpit closure UX, and validation-script coverage for policy read, alert read, ticket create/reuse, record verification, and read-only RBAC guard.

## Highlights

- Added runbook risk-alert ticket linkage persistence and read model.
- Added one-click create/reuse ticket action endpoint with dedup semantics.
- Added cockpit risk-alert row actions and linked-ticket status rendering.
- Added v0.1.11 operator validation script with required stage keys and artifact outputs.

## Upgrade Steps

1. Pull `v0.1.11` code and restart API/web services.
2. Apply migration `202603100002_create_runbook_risk_alert_ticket_links.sql`.
3. Verify new/updated APIs:
   - `GET /api/v1/ops/cockpit/runbook-templates/analytics/alerts` (with `ticket_link` field)
   - `POST /api/v1/ops/cockpit/runbook-templates/analytics/alerts/tickets`
4. Open cockpit runbook panel and validate risk-alert ticket linkage/action workflow.
5. Run v0.1.11 operator validation:
   - `bash scripts/qa-v0.1.11-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.11-operator-journey.sh --auth-user viewer --read-only`
