# CloudOps One v0.1.8

## Release Metadata

- Version: `v0.1.8`
- Release date: `2026-03-09`
- Release type: `minor`
- Git tag: `v0.1.8`
- GitHub issues: `#151, #152, #153, #154, #155`

## Executive Summary

v0.1.8 improves runbook repeatability for non-technical operators by introducing a preset library and one-click replay from execution history.

This release reduces manual parameter re-entry, improves replay traceability, and keeps policy/RBAC guardrails in the execution path.

## Highlights

- Added runbook execution preset CRUD API with audit coverage.
- Added one-click replay API with source-execution linkage.
- Added cockpit preset/apply/replay workflow for daily operators.
- Added v0.1.8 validation suite and release-gate checklist.

## Upgrade Steps

1. Pull `v0.1.8` code and apply migrations:
   - `202603090002_create_runbook_execution_presets.sql`
   - `202603090003_add_replay_source_to_runbook_template_executions.sql`
2. Restart API and web-console services.
3. Verify preset/replay endpoints:
   - `GET /api/v1/ops/cockpit/runbook-templates/presets`
   - `POST /api/v1/ops/cockpit/runbook-templates/executions/{id}/replay`
4. Validate cockpit runbook preset and replay flow using write-role account.
5. Run v0.1.8 operator validation:
   - `bash scripts/qa-v0.1.8-operator-journey.sh --auth-user admin`
   - `bash scripts/qa-v0.1.8-operator-journey.sh --auth-user viewer --read-only`
