# v0.1.5 Escalation and Failover Simulation Guide

## Purpose

This simulation provides reproducible dry-run evidence for release readiness by replaying escalation and failover scenarios against a policy budget.

It is designed for operator teams to answer one question quickly:

- If queue pressure and regional failover happen together, do we still stay inside agreed operating budgets?

## One-Command Usage

```bash
bash scripts/ops-escalation-failover-sim.sh
```

Default artifacts:

- `.run/sim/v0.1.5-escalation-failover-<RUN_ID>/summary.json`
- `.run/sim/v0.1.5-escalation-failover-<RUN_ID>/summary.md`

## Inputs

- Simulation policy: `scripts/ops-escalation-failover-policy.json`
- Optional live baseline from API escalation queue:

```bash
bash scripts/ops-escalation-failover-sim.sh --live-baseline --api-base-url http://127.0.0.1:8080 --auth-user admin
```

## Policy Model

The policy file controls:

- Baseline metrics (escalation backlog, near-breach, breached tickets, failover RTO/error rate).
- Budgets (max allowed values per metric).
- Scenario matrix (multiplier/additive factors and remediation hints).

Operators can safely customize scenario and threshold values without code changes.

## Output Interpretation

- `summary.json`: machine-readable checks, per-scenario projections, and violation entries.
- `summary.md`: operator-focused report with scenario table and recommended actions.

A violation includes:

- impacted scenario,
- metric and threshold,
- remediation hint (actionable next step).

## Assumptions

- Baseline values represent current operational posture.
- Scenario factors approximate relative stress effects, not exact physical replay.
- Failover budget uses RTO/error-rate abstractions suitable for release gating.

## Limitations

- This is a policy-driven model; it does not execute real traffic failover.
- Live-baseline mode only samples escalation queue values from API.
- Result quality depends on baseline quality and scenario tuning.

## Recommended Workflow

1. Tune `scripts/ops-escalation-failover-policy.json` to your environment.
2. Run script in CI/staging before release publish.
3. Attach `summary.json` and `summary.md` in release closure issue.
4. If violations exist, apply the hinted remediations and rerun.
