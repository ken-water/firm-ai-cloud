# CloudOps One v0.1.31 Planning (Usability + UX Polish Only)

Date: 2026-03-20
Version: v0.1
Target release: `v0.1.31`

## 1. Release Focus (Only One Priority)

This release has a single priority:

- improve usability, user experience, and visual clarity.

Explicitly excluded from `v0.1.31`:

- new business-domain capabilities,
- new API surface for feature expansion,
- workflow logic expansion unrelated to UX simplification.

## 2. Problem Statement

Current console still feels overloaded:

- users cannot quickly identify where to start from each page,
- core flow (site/room -> rack -> device -> monitoring -> alert/topology) is not represented as a simple and stable entry path across all pages,
- pages present too much mixed content by default, causing cognitive overload.

## 3. UX Target State

By end of `v0.1.31`, every main page should satisfy:

1. one clear default start area visible above the fold,
2. grouped navigation with stable mental model (menu -> sub-menu -> page tabs),
3. page content split into:
   - essential first-screen blocks,
   - advanced blocks hidden behind explicit expansion,
4. consistent visual language for status, spacing, typography, and interaction cues,
5. predictable click path from entry to action within 2-3 steps.

## 4. IA and Navigation Design Baseline

### 4.1 Left Menu (Level 1)

Use stable groups:

- `Start`
- `Assets`
- `Operations`
- `System`

Rules:

- no cross-group duplication,
- group names are action-oriented and user-language friendly,
- each page belongs to exactly one group.

### 4.2 Group Sub-menu (Level 2)

For active group, show only pages inside this group in top sub-navigation tabs.

Rules:

- max 4-6 tabs visible before overflow,
- active tab always highlighted,
- tab order follows user journey, not technical dependencies.

### 4.3 Page Section Tabs (Level 3)

Each page has section tabs for in-page navigation.

Rules:

- first tab is the default start tab,
- tabs map to concrete sections and auto-scroll/focus,
- labels use user task terms (for example: `Start`, `Assets`, `Monitoring`, `Alerts`, `Advanced`).

## 5. Page Focus Pattern (Mandatory)

For every main page:

1. **First Screen**
   - one summary sentence of page purpose,
   - one prioritized action area,
   - max 3-5 cards/blocks.
2. **Secondary Area**
   - medium-frequency tasks.
3. **Advanced Area**
   - low-frequency or expert content collapsed by default.

## 6. Visual and Interaction Standard

1. Typography hierarchy:
   - page title, section title, body text, helper text are visually distinct.
2. Spacing rhythm:
   - consistent spacing scale between cards and controls.
3. Status semantics:
   - only `ready / attention / blocking`.
4. Control consistency:
   - primary action style is unique and consistent,
   - destructive actions require stronger visual warning.
5. Empty states:
   - every empty block gives next-step guidance and direct entry button.

## 8. UX Acceptance Metrics

Minimum acceptance signals for this release:

1. New user can locate the primary start path in under 10 seconds.
2. Top 5 tasks can be reached in 2-3 clicks from menu entry.
3. First-screen visible blocks per page are capped and prioritized.
4. No page shows mixed advanced content by default without collapse.
