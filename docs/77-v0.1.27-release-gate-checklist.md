# v0.1.27 Release Gate Checklist

Date: 2026-03-20
Version: v0.1.27

## 1. Preconditions

1. API and web-console run against migrated database.
2. `v0.1.26` release is complete and synchronized.
3. Demo/stakeholder accounts are available for overview and large-screen routes.
4. `v0.1.27` scope is mapped to:
   - `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

## 2. Required Planning Evidence

1. Planning issue maps visual system, executive summary, scene playlist, and click-path outcomes.
2. Scope defines user-visible improvements for non-technical decision makers.
3. Scope defines acceptance artifact expectations for visual and large-screen validation.

## 3. Required Validation Coverage

1. Visual system baseline is validated for hierarchy consistency.
2. Executive summary hero blocks are validated for stakeholder readability.
3. Scene playlist baseline is validated for configuration and rotation behavior.
4. Click-path compression is validated for top workflows.

## 4. Required Artifacts

- `summary.json`
- `summary.md`
- `artifact-index.json`
- stage logs (`*.log`)

Release notes metadata must include:

- `- Demo acceptance artifacts: \`<summary.json path>, <artifact-index.json path>\``
- `- Role cockpit acceptance artifacts: \`<summary.json path>, <artifact-index.json path>\``

## 5. Release Gates

Before publish:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

Publish sync:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.27 --notes-file release-notes/v0.1.27.md
make release-publish-dry VERSION=0.1.27
make release-publish VERSION=0.1.27
make release-check VERSION=0.1.27
```
