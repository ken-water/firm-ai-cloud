# CloudOps One v0.1.32 Planning (IA + Visual Adoption)

Date: 2026-03-23
Version: v0.1
Target release: `v0.1.32`

## 1. Release Focus

`v0.1.32` has one focus:

- make existing capabilities acceptable and efficient for users through better information architecture and page experience.

Explicitly excluded from `v0.1.32`:

- new domain-feature expansion,
- broad API-surface expansion unrelated to UX/IA adoption.

## 2. Problem Statement

Current product bottleneck is not capability coverage, but adoption friction:

- users still need too much interpretation to find where to start,
- pages can feel function-rich but cognitively dense,
- interface attractiveness and hierarchy are not yet at a level that improves trust in first-time trials.

## 3. Target Outcome

By end of `v0.1.32`:

1. new users can find top tasks quickly with a stable, traditional menu pattern,
2. every core page communicates one clear purpose and one obvious first action,
3. visual quality improves materially without changing core task paths,
4. release evidence includes measurable usability outcomes.

## 4. UX Strategy: Compatibility First, Modernization Second

### 4.1 Traditional Menu Compatibility (Mandatory)

Use a stable, predictable IA baseline first:

- level-1 menu limited to 5-6 user-language entries,
- level-2 grouped by task context, not backend module names,
- level-3 page tabs for in-page task switching.

This ensures broad user acceptance even for conservative enterprise users.

### 4.2 Modern Visual Uplift (Non-disruptive)

Modernization must not break familiarity:

- improve typography/spacing/status hierarchy,
- improve card and action affordance clarity,
- refine visual rhythm and empty-state guidance,
- preserve click-path and task sequence.

## 5. Page Contract (Mandatory)

Each core page must follow one contract:

1. purpose row: what this page is for,
2. start area: first action block,
3. action lane: high-frequency operations,
4. detail lane: data table/detail exploration,
5. advanced area: folded by default.

## 6. Usability Acceptance Metrics

Minimum measurable signals:

1. top 5 tasks can be located in under 10 seconds,
2. top 5 tasks can be completed in <= 3 clicks from menu entry,
3. first-screen blocks per core page are capped at 3-5,
4. every core page exposes a visible "start action" without scrolling.
