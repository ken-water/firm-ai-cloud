# CloudOps One v0.1.9 Planning (Runbook Analytics and Risk Visibility)

Date: 2026-03-09  
Version: v0.1  
Target release: `v0.1.9`

## 1. Planning Goal

v0.1.9 closes the highest-priority post-v0.1.8 operations gap: teams can execute/replay runbooks quickly, but they still lack clear visibility into quality trends, failure hotspots, and remediation priorities.

Target outcomes:

1. Operators can see runbook success/failure trends in a bounded time window.
2. Operators can quickly identify failure hotspots (failed step, output, remediation hint).
3. Cockpit runbook panel exposes analytics in a no-code workflow.
4. Release gate includes analytics-focused operator journey validation.

## 2. Scope

In scope:

- Runbook analytics summary endpoint (`totals`, `template breakdown`, `top failed steps`).
- Runbook failure feed endpoint with filter/pagination for triage.
- Web-console runbook analytics panel and failure hotspot view.
- v0.1.9 validation script and release gate checklist.

Out of scope:

- Cross-workspace BI/reporting pipeline.
- Long-term warehouse export or external observability sink.
- Auto-remediation recommendations based on ML ranking.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Analytics payload too large in high-volume clusters | slow cockpit load and noisy UX | enforce bounded day range and pagination defaults |
| Failure diagnostics too raw for non-technical users | low remediation efficiency | expose failed step id/output plus explicit remediation hint |
| Dashboard trust gap from stale data | operators ignore panel signals | include generated timestamp and manual refresh controls in UI |
