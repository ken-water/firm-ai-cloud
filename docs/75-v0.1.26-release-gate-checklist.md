# v0.1.26 Release Gate Checklist

Date: 2026-03-19
Version: v0.1.26

## 1. Preconditions

1. API and web-console run against migrated database.
2. `v0.1.25` release is complete and synchronized.
3. Role test accounts (`operator/admin/network/business`) are available.
4. `v0.1.26` scope is mapped to:
   - `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

## 2. Required Planning Evidence

1. Planning issue maps role cockpit, AI follow-up, and ownership continuity outcomes.
2. Scope defines role-specific user-visible simplifications.
3. Scope defines acceptance artifact expectations for role-cockpit validation.

## 3. Required Validation Coverage

1. Role cockpit quick scene switch/apply flow is validated.
2. AI follow-up intent cards are validated against deterministic evidence mapping.
3. Topology and ownership joint visibility is validated for follow-up clarity.
4. Role-cockpit acceptance artifacts are generated and linked in release issue.

## 4. Required Artifacts

- `summary.json`
- `summary.md`
- `artifact-index.json`
- stage logs (`*.log`)

Release notes metadata must include:

- `- Demo acceptance artifacts: \`<summary.json path>, <artifact-index.json path>\``

## 5. Release Gates

Before publish:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

Publish sync:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.26 --notes-file release-notes/v0.1.26.md
make release-publish-dry VERSION=0.1.26
make release-publish VERSION=0.1.26
make release-check VERSION=0.1.26
```
