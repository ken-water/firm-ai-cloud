# v0.1.9 Release Gate Checklist

Date: 2026-03-09  
Version: v0.1.9

This checklist defines the operator-facing release gate for `v0.1.9`.

## 1. Preconditions

1. API and web-console are running against a migrated database.
2. At least one write-capable account and one read-only account are available.
3. Release note metadata includes closed GitHub issue list:
   - `- GitHub issues: \`#157, #158, #159, #160, #161\``
   - release closure issue `#162` is closed after publish evidence is posted.

## 2. One-Command Validation

Write-capable scenario:

```bash
bash scripts/qa-v0.1.9-operator-journey.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user admin
```

Read-only scenario:

```bash
bash scripts/qa-v0.1.9-operator-journey.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user viewer \
  --read-only
```

## 3. Required Stage Coverage

The summary must include these stage keys:

1. `baseline_read`
2. `simulate_execute`
3. `replay_execute`
4. `analytics_summary`
5. `failure_feed`
6. `records_verify`

## 4. Required Artifacts

Each run must generate:

1. `summary.json`
2. `summary.md`
3. `artifact-index.json`
4. Per-stage logs: `*.log`

Default output directory pattern:

- `.run/qa/v0.1.9-journey-<RUN_ID>/`

## 5. Failure Localization Rule

If `overall=fail`, `summary.md` must contain:

1. failed stage key,
2. endpoint path,
3. remediation pointer,
4. stage diagnostics.

Release cannot proceed until all failed stages are resolved or explicitly deferred in release closure issue.

## 6. Release Gate Commands

Before publishing:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

After release note/changelog finalization:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.9 --notes-file release-notes/v0.1.9.md
make release-publish-dry VERSION=0.1.9
make release-publish VERSION=0.1.9
make release-check VERSION=0.1.9
```
