# CloudOps One v0.1.1 Validation Suite

Date: 2026-03-05
Version: v0.1.1

## 1. Goal

Validate the non-expert daily operations flow introduced in v0.1.1:

- Playbook catalog + dry-run/confirm execution safety path
- Daily prioritized cockpit queue and action entry points
- Topology edge diagnostics context and guided troubleshooting panel
- RBAC read-only restrictions for viewer role

## 2. Automated Entry Point

Run:

```bash
bash scripts/qa-v0.1.1.sh
```

Output artifacts:

- `.run/qa/v0.1.1-<timestamp>/summary.json`
- `.run/qa/v0.1.1-<timestamp>/summary.md`
- per-step logs under the same folder

## 3. Scenario Matrix

| Scenario | Path Type | What is validated | Source |
| --- | --- | --- | --- |
| Playbook dry-run + execute gate | Happy + deny | Param schema checks, high-risk confirmation flow, replay endpoint, RBAC write mapping | `services/api/src/playbooks.rs`, `services/api/src/auth.rs` |
| Cockpit queue ranking | Happy | Deterministic ordering and rationale metadata across alerts/tickets/sync/stream stale | `services/api/src/cockpit.rs` |
| Topology diagnostics contract | Happy + deny | Edge diagnostics trend/alerts/changes/impacted hints and scope guard behavior | `services/api/src/topology.rs` |
| Viewer read-only enforcement | Deny | Viewer blocked from playbook execution while retaining read access | `scripts/test-rbac-policy.sh`, `services/api/src/auth.rs` |
| Web UI consistency | Happy | Build/type checks, i18n parity, guardrail checks | `apps/web-console/package.json` -> `check:ui` |

## 4. CI Reproducibility

The validation checks are reproducible in CI through existing mandatory gates:

- `cargo test -p api`
- `cd apps/web-console && npm run check:ui`

Optional integration matrix (requires running API):

- `bash scripts/test-rbac-policy.sh`
