# CloudOps One v0.1.8 Planning (Runbook Standardization and Replay)

Date: 2026-03-09  
Version: v0.1  
Target release: `v0.1.8`

## 1. Planning Goal

v0.1.8 closes the highest-priority post-v0.1.7 usability gap: operators still re-enter runbook parameters manually and cannot replay known-good executions in one step.

Target outcomes:

1. Operators can save reviewed runbook parameter presets and reuse them safely.
2. Operators can replay previous executions with policy/RBAC guardrails.
3. Cockpit UX clearly exposes preset apply and replay actions for low-friction daily operations.
4. Release gate includes preset/replay operator-journey validation.

## 2. Scope

In scope:

- Runbook execution preset data model and CRUD API.
- Replay endpoint based on execution history (`executions/{id}/replay`).
- Replay audit metadata linking source execution id.
- Web-console preset apply + replay workflow.
- v0.1.8 validation script and release gate checklist.

Out of scope:

- Distributed/multi-node replay scheduler.
- Automatic replay recommendation engine.
- Cross-template preset conversion.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Preset drift from template changes | replay failures and confusion | validate params against current template schema on apply and replay |
| Unsafe replay in live mode | production risk | enforce existing execution policy and live allowlist during replay |
| Preset sprawl | operator choice overload | support scoped list/filter by template and explicit ownership metadata |
