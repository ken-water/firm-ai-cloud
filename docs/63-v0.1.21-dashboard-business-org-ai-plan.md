# CloudOps One v0.1.21 Planning (Dashboard Studio + Business/Org/AI Baseline)

Date: 2026-03-18
Version: v0.1
Target release: `v0.1.21`

## 1. Problem Statement

`v0.1.20` closed the owner/escalation action loop, but product usability still has major gaps:

- module entry experience is still too dense and not dashboard-first,
- monitoring and CMDB are not yet business-centric enough for direct decision use,
- cross-department workflow and approval design needs a stronger baseline,
- AI access exists conceptually but not yet as evidence-backed operator value path.

## 2. Goals

1. Establish a configurable dashboard foundation so users can get role-specific value at first glance.
2. Define business-centric monitoring/CMDB data contracts as mandatory follow-up scope.
3. Define organization-aware workflow/approval baseline for cross-team execution.
4. Define evidence-backed AI analytics entry baseline (answer + source + time range).
5. Lock strategy-to-release alignment as a required planning and release discipline.

## 3. Scope

- add `v0.1.21` planning and release-gate docs aligned with strategy baseline,
- create issue-first execution map for dashboard/business/org/AI foundation,
- define validation stages for dashboard baseline and read-only/write guard paths.

## 4. Out of Scope

- full dashboard studio implementation in one release,
- complete multi-department approval engine rollout,
- full AI agent execution orchestration,
- all SaaS connector integrations in a single milestone.

## 5. Strategic Alignment

This cycle is mandatory-aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.21`:

1. customizable operations dashboard system,
2. business-centric operations model,
3. organization and workflow model,
4. evidence-backed AI value layer.

  - `make release-check VERSION=0.1.21`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Dashboard scope becomes visual-only and not decision-ready | low operational value | enforce evidence widgets and owner/action visibility requirements |
| Business model complexity slows delivery | release delay | ship baseline schema/contract first, then iterate connectors |
| Workflow over-design increases SMB friction | poor adoption | provide simple default templates and progressive advanced options |
| AI without evidence reduces trust | low usage | require source/time-range references in response contract |
