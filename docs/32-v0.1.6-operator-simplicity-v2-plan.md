# CloudOps One v0.1.6 Planning (Operator Simplicity v2)

Date: 2026-03-07
Version: v0.1
Target release: `v0.1.6`

## 1. Planning Goal

v0.1.6 focuses on reducing operator decision overhead after v0.1.5 continuity foundations.

Target outcomes:

1. Non-technical operators can start from a safe profile without manual policy tuning.
2. Cockpit provides deterministic next actions and scheduling guidance instead of operator guesswork.
3. Handover and restore evidence quality are enforced with explicit overdue/compliance signals.
4. Release validation captures full operator journey with stage-level diagnostics.

## 2. Scope

In scope:

- Setup profile presets with preview/apply and audit trail.
- Cockpit next-best-action assistant.
- Change calendar reservation and conflict-free slot recommendation.
- Handover overdue reminder and ownership quality gates.
- Restore evidence SLA compliance scorecard.
- One-click runbook templates with preflight and evidence.
- Extended validation suite and release closure evidence.

Out of scope:

- Autonomous remediation without operator confirmation.
- Full external ITSM migration.
- Real traffic orchestration across active-active multi-region runtime.

## 4. Milestone Plan

### M1: Guided Setup and Decision Support

- Target issues: `#136 #137 #138`
- Exit criteria:
  - operators can start from safe presets;
  - cockpit surfaces deterministic next actions.

### M2: Scheduling and Continuity Quality Gates

- Target issues: `#139 #140 #141`
- Exit criteria:
  - risky operations can be scheduled with reservation guardrails;
  - handover/evidence overdue gaps are explicit and actionable.

### M3: No-Code Execution and Release Closure

- Target issues: `#142 #143 #144`
- Exit criteria:
  - runbook templates are executable and auditable;
  - validation and release evidence gates are complete.

## 5. Definition of Done (v0.1.6)

- Issues `#136` to `#144` are closed (or explicitly deferred with reason in `#144`).
- Mandatory checks pass:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
- Validation evidence is attached in release closure:
  - operator scenario summary artifacts
  - simulation/compliance summary artifacts
- Release automation gates pass:
  - `make release-publish-dry VERSION=0.1.6`
  - `make release-publish VERSION=0.1.6`
  - `make release-check VERSION=0.1.6`

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Too many cockpit suggestions | Decision fatigue | Keep deterministic ranking and strict top-N display |
| Preset misfit for real environments | Misconfiguration risk | Require preview diff + audit and easy rollback |
| Reservation workflow adds friction | Slow change execution | Support quick auto-slot recommendation + one-click reserve |
| Compliance gates become noisy | Alert fatigue | Add SLA thresholds and scoped reminder exports |
