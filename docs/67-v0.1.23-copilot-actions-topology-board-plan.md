# CloudOps One v0.1.23 Planning (Copilot Guided Actions + Topology Board + Handoff Readiness)

Date: 2026-03-18
Version: v0.1
Target release: `v0.1.23`

## 1. Problem Statement

`v0.1.22` delivered dashboard role templates, business topology baseline, AI intent presets, and one-screen handoff view. Remaining SMB adoption gaps are:

- AI answers are evidence-backed but still mostly read-only summaries,
- topology view is service-level but lacks board-style action perspective,
- handoff summary is visible but not yet aligned with explicit ready/at-risk states,
- operators still need clearer guidance from insight to safe action.

## 2. Goals

1. Deliver copilot guided-action baseline with explicit read-only/write-safe boundaries.
2. Deliver topology board v1 for service/network risk and ownership focus.
3. Deliver handoff readiness state model (ready/at-risk/blocking) for shift decisions.
4. Keep release process issue-first and release-sync gated.

## 3. Scope

- planning and release-gate docs for `v0.1.23`,
- issue-first execution map for copilot/topology/handoff tracks,
- deterministic API contracts and cockpit-visible UX,
- validation for read-only/write-guard behavior.

## 4. Out of Scope

- autonomous multi-step agent execution,
- full topology editing studio,
- multi-tenant action policy orchestration,
- external ITSM bi-directional sync expansion.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.23`:

1. AI-guided operator value path,
2. business/network topology decision board,
3. handoff readiness confidence,
4. safe action guardrails for SMB teams.

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| guided actions may overstep safety boundaries | unsafe mutation risk | enforce action policy metadata and explicit write guards |
| topology board can become noisy | poor readability | keep priority/risk/owner-first ordering and hide low-value details |
| handoff readiness can drift from reality | low trust | derive readiness from deterministic source contracts and timestamped evidence |
| release drift from issue map | governance regression | keep issue-first closure and release-sync gate mandatory |
