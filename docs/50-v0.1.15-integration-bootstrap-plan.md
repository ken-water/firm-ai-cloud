# CloudOps One v0.1.15 Planning (Integration Productization and Guided Bootstrap)

Date: 2026-03-15  
Version: v0.1  
Target release: `v0.1.15`

## 1. Planning Goal

v0.1.15 starts closing the next maturity gap after v0.1.14:
the product can now repair runbook-risk notification readiness, but operators still need too much product knowledge to connect and bootstrap the broader notification and ticketing paths that make daily operations actually usable.

Target outcomes:

1. Operators can see an opinionated integration catalog with recommended bootstrap defaults.
2. Notification and ticket-follow-up foundations can be applied from a guided product flow instead of scattered low-level pages.
3. Bootstrap apply remains idempotent and safe for repeated use.
4. Release gate validates discover -> apply -> verify -> read-only guard.

## 2. Scope

In scope:

- integration catalog and bootstrap recommendation API for operator-facing defaults,
- guided bootstrap apply flow for notification and ticket-follow-up prerequisites,
- cockpit/setup workspace for integration bootstrap and gap visibility,
- v0.1.15 validation script and release gate checklist.

Out of scope:

- deep vendor-specific ITSM adapters beyond current internal ticket model,
- full marketplace/plugin SDK,
- multi-tenant integration governance redesign.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Bootstrap recommendations are too generic | operators still need product experts | keep catalog opinionated around default enterprise paths first |
| Apply flow becomes unsafe | duplicate or conflicting integration config | use create-or-enable semantics and explicit result evidence |
| Workspace scope grows too wide | UX becomes another admin dumping ground | keep v0.1.15 focused on a short operator bootstrap sequence |
