# v0.1.32 Release Gate Checklist

Date: 2026-03-23
Version: v0.1.32

## 1. Preconditions

1. `v0.1.31` release is complete and synchronized.
2. `v0.1.32` scope remains UX/IA-only.
3. Planning baseline is aligned with:
   - `docs/86-v0.1.32-ia-and-visual-adoption-plan.md`

## 2. Required Planning Evidence

1. task-first menu taxonomy is explicit and testable,
2. page composition contract is explicit and reusable,
3. usability metrics are measurable and release-checkable.

## 3. Required Validation Coverage

1. top 5 tasks are discoverable through menu hierarchy,
2. top 5 tasks are reachable within <= 3 clicks,
3. core pages expose clear first action without scrolling,
4. first-screen block count is capped and advanced content stays folded by default,
5. visual hierarchy consistency is improved across major pages.

## 4. Required Artifacts

For usability acceptance evidence (`v0.1.32`):

- `summary.json`
- `summary.md`
- `artifact-index.json`
- stage logs (`*.log`)

Release notes metadata must include all existing acceptance artifact fields plus:

- `- UX acceptance artifacts: \`<summary.json path>, <artifact-index.json path>\``
- `- Usability acceptance artifacts: \`<summary.json path>, <artifact-index.json path>\``

## 5. Release Gates

Before publish:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

Publish sync:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.32 --notes-file release-notes/v0.1.32.md
make release-publish-dry VERSION=0.1.32
make release-publish VERSION=0.1.32
make release-check VERSION=0.1.32
```
