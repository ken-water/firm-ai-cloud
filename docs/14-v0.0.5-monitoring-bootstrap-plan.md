# CloudOps One v0.0.5 Monitoring Bootstrap Plan

Date: 2026-03-03

This document tracks the issue-first implementation plan for v0.0.5.

## 1. Problem Inventory (Current State)

1. There is no dedicated monitoring source registry for Zabbix server/proxy onboarding.
2. Monitoring API contracts are not ready for frontend integration.
3. Web-console has no monitoring source operation surface yet.
4. Real-time alert transport baseline is still missing.

## 2. Iteration Backlog

| Iteration | GitHub Issue | Focus |
| --- | --- | --- |
| I1 | `#28` | Monitoring source registry + probe APIs |
| I2 | `#29` | Monitoring overview/layer API contracts |
| I3 | `#30` | Web-console monitoring source UX |
| I4 | `#31` | SSE stream baseline for real-time alerts |
| I5 | `#32` | Bundled Zabbix server/proxy/local-agent one-click deployment |

## 3. Iteration Rules

1. Keep each iteration independently releasable.
2. Preserve manual-trigger CI policy and local validation-first workflow.
3. Keep docs and release-facing text in English.
4. Ensure RBAC mapping and audit traceability for new protected routes.

## 4. I1 Definition of Done (`#28`)

- Monitoring source schema is available via migration.
- API endpoints exist for source list/create/probe.
- RBAC permissions and mapping are in place (`monitoring.sources.read/write`).
- Operator/viewer role behavior is covered by scripts/tests.
- Workspace checks pass.

## 9. I4 Completion Notes (`#31`)

Status: Completed on 2026-03-03.

Delivered updates:

1. Added SSE baseline endpoint:
   - `GET /api/v1/streams/sse`
2. Added stream envelope contract:
   - `event_type`
   - `scope` (`site`, `department`, `severity`)
   - `timestamp`
   - `payload`
3. Added stream filters:
   - `site`
   - `department`
   - `severity` (`all`, `critical`, `warning`, `info`)
4. Added scope enforcement baseline:
   - RBAC route mapping via `monitoring.sources.read`
   - non-admin subscriptions require at least one scope filter (`site`/`department`)
   - optional auth-scope headers (`x-auth-site`, `x-auth-department`) deny out-of-scope subscriptions with explicit errors
5. Added stale-state event semantics:
   - `stream.heartbeat`
   - `stream.stale`
   - `stream.recovered`
6. Added polling baseline to emit monitoring-sync alert events:
   - `alert.monitoring_sync`
   - plus bootstrap `alert.test` event for contract verification.

Validation:

- `cargo check -p api`
- `cargo test -p api`

Manual verification checklist:

1. Start API service and ensure RBAC is enabled.
2. Subscribe as operator/viewer:
   - `curl -N -H "x-auth-user: <user>" "http://127.0.0.1:8080/api/v1/streams/sse?site=dc-a&severity=all"`
3. Confirm initial events are received:
   - `stream.connected`
   - `alert.test`
4. Trigger a monitoring sync job (for any eligible asset) and verify `alert.monitoring_sync` arrives.
5. Keep stream idle for stale window and verify `stream.stale`, then trigger another sync job and verify `stream.recovered`.
6. Verify out-of-scope denial:
   - `curl -N -H "x-auth-user: <user>" -H "x-auth-site: dc-a" "http://127.0.0.1:8080/api/v1/streams/sse?site=dc-b"`
   - expect HTTP `403` with explicit scope-denied message.
