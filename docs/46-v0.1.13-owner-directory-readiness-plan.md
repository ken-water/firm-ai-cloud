# CloudOps One v0.1.13 Planning (Owner Directory and Notification Readiness)

Date: 2026-03-11  
Version: v0.1  
Target release: `v0.1.13`

## 1. Planning Goal

v0.1.13 closes the next highest-priority operator gap after v0.1.12:
runbook-risk tickets can now route and notify, but operators still need a simple, configurable way to define who owns follow-up and whether notification coverage is actually ready.

Target outcomes:

1. Runbook-risk owner routing is configurable instead of hardcoded.
2. Operators can see notification-readiness gaps before they depend on automatic follow-up.
3. Cockpit/setup surfaces show configured owner names and missing coverage clearly.
4. Release gate validates configure -> route -> readiness verify -> guard.

## 2. Scope

In scope:

- configurable owner directory and runbook-risk routing rules,
- notification-readiness coverage summary and gap detection,
- cockpit/setup visibility for owner and readiness state,
- v0.1.13 validation script and release gate checklist.

Out of scope:

- full on-call scheduling engine,
- generic HR or org-chart management,
- multi-tenant notification redesign across unrelated modules.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Owner config is too abstract | operators still cannot map alerts to real owners | keep owner directory minimal and operator-readable |
| Readiness signal is too noisy | users ignore missing coverage warnings | summarize by actionable gap categories with deterministic states |
| Setup UX becomes too technical | non-technical operators still need engineering help | expose opinionated defaults and simple labels first |
