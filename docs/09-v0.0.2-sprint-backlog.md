# CloudOps One v0.0.2 Sprint Backlog

Version: v1.0  
Date: 2026-03-02  
Target release: `v0.0.2`  
Sprint length: 2 weeks

## 1. Sprint Goal

Deliver the first production-usable CMDB operation loop:

- configurable asset relationships (device-business-owner-external system)
- auto discovery baseline (at least one active discovery source)
- onboarding/offboarding change detection
- automatic notifications for CMDB change events
- English-only release-quality docs and API examples

## 2. In Scope

- CMDB relationship model and APIs
- discovery jobs and candidate queue
- change event generation and notification dispatch
- minimal CMDB UI pages for relationship and discovery operations
- backend and frontend validation for release gate

## 3. Out of Scope (v0.0.2)

- full topology big-screen implementation
- advanced ticket workflow engine
- full enterprise SSO integration
- multi-region/high-availability deployment features

## 4. Definition of Done for v0.0.2

1. All P0 issues in this sprint are completed.
2. API + UI paths are demo-ready with sample data.
3. `cargo check --workspace` and frontend `npm run build` pass.
4. Release notes and changelog are complete in English.
5. At least one end-to-end flow is verified:
   - discovery detects a new asset
   - change event generated
   - notification delivered

## 6. Detailed Acceptance Criteria by Issue

### V020-001: Add CMDB relationship schema and migration (P0)

- Add `asset_relations` table with relation type, source, timestamps.
- Add uniqueness guard for duplicated relation edges.
- Add index for `src_asset_id`, `dst_asset_id`, and `relation_type`.
- Migration is forward-compatible and idempotent.

### V020-002: Implement relationship CRUD and query APIs (P0)

- `POST /api/v1/cmdb/relations`
- `GET /api/v1/cmdb/relations?asset_id=...`
- `DELETE /api/v1/cmdb/relations/{id}`
- Validation: no self-loop unless explicitly allowed by type.
- Error model follows existing API conventions.

### V020-003: Add relation graph endpoint for asset detail (P0)

- `GET /api/v1/cmdb/assets/{id}/graph`
- Returns one-hop upstream/downstream relations.
- Includes minimal node metadata (`id`, `name`, `asset_class`, `status`).

### V020-004: Add discovery job and candidate tables + APIs (P0)

- Add `discovery_jobs` and `discovery_candidates` schema.
- APIs:
  - `POST /api/v1/cmdb/discovery/jobs`
  - `GET /api/v1/cmdb/discovery/jobs`
  - `POST /api/v1/cmdb/discovery/jobs/{id}/run`
  - `GET /api/v1/cmdb/discovery/candidates`
- Candidate record stores fingerprint and raw payload.

### V020-005: Implement first discovery source adapter (P0)

- Discovery source: Zabbix host list pull.
- Dedup policy baseline: `hostname + ip`.
- Writes unmatched hosts into candidate queue.
- Discovery run logs status and error details.

### V020-006: Implement asset change detector (P0)

- Emit event types:
  - `asset.new_detected`
  - `asset.offboarded_suspected`
  - `asset.profile_changed`
- Offboarded policy: missing in N consecutive discovery runs (configurable).
- Event records are queryable by asset and time range.

### V020-007: Add notification channels/templates/subscriptions (P0)

- Add channel config (email/webhook for MVP).
- Add template model by event type.
- Add subscription scope by site/department/event type.

### V020-008: Dispatch notifications for CMDB events (P0)

- On event creation, notification task is created and dispatched.
- Retry with exponential backoff on failure.
- Delivery log records receiver, status, and response code.

### V020-009: CMDB relation panel UI (P0)

- Asset detail page shows related assets grouped by relation type.
- Supports add/remove relation action.
- All UI text in `en-US` locale bundle.

### V020-010: Discovery jobs + candidate review UI (P0)

- Discovery job list and manual run action.
- Candidate queue with approve/reject action.
- Approve action creates or merges asset.

### V020-011: Notification rule/subscription management UI (P1)

- Add event subscription list and create form.
- Add channel selection and target input.
- Basic validation messages in English.

### V020-012: Integration tests for CMDB-discovery-notification loop (P0)

- API tests cover relation CRUD and discovery job lifecycle.
- Worker tests cover event generation and notification dispatch.
- One end-to-end test demonstrates full CMDB event path.

### V020-013: Update quickstart and API docs (P0)

- Document new discovery and relation APIs.
- Add sample curl commands for core flow.
- Add troubleshooting section for common failures.

### V020-014: Prepare `v0.0.2` release notes/changelog (P0)

- `release-notes/v0.0.2.md` created from template.
- `CHANGELOG.md` updated.
- Validate notes via `scripts/validate-release-note.sh`.

## 9. Risk and Mitigation

| Risk | Impact | Mitigation |
|---|---|---|
| Discovery source instability | Delayed candidate generation | Add retry + timeout + fallback logging |
| Event noise from frequent changes | Notification fatigue | Add dedup window and severity filter |
| Schema changes break existing API | Regression risk | Add migration tests and backward-compat checks |
| UI scope creep | Sprint delay | Keep V020-011 as P1 and cut if needed |
