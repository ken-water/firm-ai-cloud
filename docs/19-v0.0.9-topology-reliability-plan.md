# CloudOps One v0.0.9 Planning (Topology + Reliability)

Date: 2026-03-05
Version: v0.1
Target release: `v0.0.9`

## 1. Planning Goal

v0.0.9 focuses on two outcomes:

1. Deliver the first usable topology surface (API + web console workspace).
2. Improve release-grade reliability visibility for stream/worker/benchmark/DR paths.

## 2. Pre-flight Completed Before v0.0.9 Build

- CI startup hardening for `auth-rbac-integration` in `.github/workflows/ci.yml`:
  - build API binary before service startup
  - run `target/debug/api` directly
  - increase health-check window and fail fast on early process exit

This pre-flight fix removes known false-negative startup timeouts and stabilizes subsequent v0.0.9 work.

## 3. Scope

In scope:

- Topology API baseline from CMDB relations and monitoring health.
- Topology web workspace and drill view.
- SSE lag observability endpoint and stream lag summaries.
- Monitoring-sync worker burst freshness improvements.
- Benchmark threshold gate and CI optional benchmark gate job.
- DR quarterly drill automation and report artifact generation.
- Release artifacts for `v0.0.9`.

Out of scope:

- Full topology editor (manual graph mutation workflow).
- Multi-region disaster failover.
- Auto-trigger CI on push/PR (manual policy remains).

## 5. Milestone Plan

### M1: Platform Observability and Perf Guardrails

- Target issues: `#66 #67 #68 #69`
- Exit criteria:
  - SSE lag summary is queryable and stable under burst profile.
  - benchmark threshold gate can fail regressions deterministically.
  - manual CI can optionally run benchmark gate.

### M2: Topology Baseline Delivery

- Target issues: `#64 #65`
- Exit criteria:
  - topology API returns stable node/edge graph payload by scope.
  - web topology workspace supports basic operational drill-down without blocking existing pages.

### M3: Operational Readiness and Release Closure

- Target issues: `#70 #71`
- Exit criteria:
  - DR drill script produces auditable report artifacts.
  - release artifacts validated and governance checklist complete.

## 6. Definition of Done (v0.0.9)

- All issues `#64` to `#71` are closed.
- Mandatory checks pass:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
- Benchmark + stream reports are attached/linked in release validation notes.
- Release note and changelog are complete in English and version-consistent.

## 7. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Topology payload grows too large | Slow API/UI rendering | Add pagination/windowing and limit defaults in API contract |
| SSE lag remains high under burst | Poor realtime experience | Tune worker cadence and expose lag metrics for iterative thresholding |
| Benchmark gate flakes in CI | False failure and release delay | Keep gate optional in manual CI and tune retries/timeouts with artifact review |
| DR script environment drift | Drill failure unrelated to product | Add explicit precheck diagnostics and report root causes |
