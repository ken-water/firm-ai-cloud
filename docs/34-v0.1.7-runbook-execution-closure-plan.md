# CloudOps One v0.1.7 Planning (Runbook Execution Closure)

Date: 2026-03-09  
Version: v0.1  
Target release: `v0.1.7`

## 1. Planning Goal

v0.1.7 closes the highest-priority v0.1.6 gap: runbook templates are currently simulation-only.

Target outcomes:

1. Operators can explicitly control runbook execution policy (`simulate_only` vs `hybrid_live`).
2. At least one runbook template (`dependency-check`) supports real guarded live execution.
3. Cockpit UI clearly exposes policy, execution mode, and runtime summary.
4. Release gate includes scenario validation for policy + simulate/live paths.

## 2. Scope

In scope:

- Runbook execution policy table and API.
- Runbook execute request `execution_mode` support (`simulate|live`).
- Live adapter baseline for `dependency-check` via bounded TCP probe.
- Execution record schema extension (`execution_mode`, `runtime_summary`).
- Web-console policy editing and mode selection UX.
- v0.1.7 validation script and release gate checklist.

Out of scope:

- Real service restart/rollback orchestration against production infrastructure.
- Arbitrary shell command execution from runbook payload.
- Multi-node distributed execution worker.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Live path misused for unsafe operations | production risk | keep adapter fixed-scope (`dependency-check`) + policy allowlist + timeout bounds |
| Policy misconfiguration blocks operators | workflow interruption | default `simulate_only`, clear policy UI, read-path always available |
| Live probe false negatives in unstable network | noisy failures | runtime summary includes timeout/latency context and remediation hints |
