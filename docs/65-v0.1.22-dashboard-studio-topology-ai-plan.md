# CloudOps One v0.1.22 Planning (Dashboard Studio v1 + Business Topology + AI Copilot Baseline)

Date: 2026-03-18
Version: v0.1
Target release: `v0.1.22`

## 1. Problem Statement

`v0.1.21` completed baseline contracts for module cockpit, business, org, and AI evidence. The next gap is productized usability for SMB operators:

- dashboard is configurable but not yet role/template driven,
- topology is visible but still not business-outcome oriented,
- AI entry exists but not yet closed-loop with module-specific intents,
- operator handoff still needs one-screen executive and action view.

## 2. Goals

1. Deliver dashboard studio v1 with role-first templates and widget composition baseline.
2. Deliver business-centric topology snapshot baseline for site/service risk understanding.
3. Deliver AI copilot v1 intents for monitoring/CMDB/workflow/ticket summaries with evidence continuity.
4. Improve one-screen operator handoff readability for SMB decision speed.
5. Keep release discipline fully issue-first and release-sync gated.

## 3. Scope

- planning and release-gate docs for `v0.1.22`,
- issue-first execution map for dashboard/topology/AI/handoff,
- deterministic API read models and cockpit-facing UX baseline,
- validation coverage for read-only and write-capable guard paths.

## 4. Out of Scope

- full drag-and-drop enterprise dashboard builder,
- complete topology animation engine and auto-layout optimizer,
- autonomous AI execution/write pipeline,
- multi-tenant isolation re-architecture.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.22`:

1. role-based dashboard customization,
2. business-outcome topology visibility,
3. evidence-backed AI value entry,
4. SMB-first one-screen operation clarity.

## 7. Definition of Done

- `#235` to `#240` closed (or explicitly deferred in release issue),
- dashboard/topology/AI/handoff baselines are user-visible in cockpit,
- release gates pass before publish:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
  - `make release-publish-dry VERSION=0.1.22`
  - `make release-publish VERSION=0.1.22`
  - `make release-check VERSION=0.1.22`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| dashboard studio grows beyond v1 scope | release delay | enforce template + contract-first baseline only |
| topology signal overload | poor operator readability | prioritize risk/owner/action and hide low-value noise by default |
| AI summary trust gap | low adoption | require evidence fields and stable source references in all responses |
| handoff panel duplication | UX clutter | consolidate into single summary contract with deterministic ordering |
