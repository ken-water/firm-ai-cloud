# CloudOps One v0.1.11 Planning (Runbook Risk-to-Ticket Closure)

Date: 2026-03-10  
Version: v0.1  
Target release: `v0.1.11`

## 1. Planning Goal

v0.1.11 closes the next highest-priority operator gap after v0.1.10:
proactive runbook risk alerts are visible, but operators still need a one-click accountable closure path.

Target outcomes:

1. Operators can convert runbook risk alerts into tracked tickets without leaving cockpit.
2. Risk alerts expose ticket linkage status to avoid duplicate or lost follow-up work.
3. Ticket creation path is RBAC-safe, auditable, and deduplicated.
4. Release gate includes explicit risk-alert-to-ticket journey validation.

## 2. Scope

In scope:

- Runbook risk alert ticket linkage persistence and read model.
- One-click create/reuse ticket action endpoint for risk alerts.
- Web-console risk-alert action panel and linkage status rendering.
- v0.1.11 validation script and release gate checklist.

Out of scope:

- External IM/SMS/Email notification fan-out for runbook risk alerts.
- Automatic ticket assignment optimization by team calendar.
- Multi-tenant portfolio-level risk scoring dashboard.

## 6. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Ticket dedup key is too broad | unrelated risks collapse into one ticket | bind dedup key to template + window + mode + open-state checks |
| Ticket dedup key is too narrow | duplicate ticket noise | create/reuse endpoint with deterministic lookup and response flag |
| Non-technical users misread risk context | slow or wrong remediation | show severity, failure-rate context, and recommended action in same panel |
