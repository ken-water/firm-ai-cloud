# v0.1.20 Release Gate Checklist

Date: 2026-03-17
Version: v0.1.20

## 1. Preconditions

1. API and web-console run against migrated database.
2. `v0.1.19` daily ops owner/closure APIs are available.
3. One write-capable and one read-only account are available.

## 2. One-Command Validation

Write-capable scenario:

```bash
bash scripts/qa-v0.1.20-owner-escalation-loop.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user admin
```

Read-only scenario:

```bash
bash scripts/qa-v0.1.20-owner-escalation-loop.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user viewer \
  --read-only
```

## 3. Required Stage Coverage

1. `daily_briefing_read`
2. `owner_assignment_apply`
3. `owner_assignment_verify`
4. `escalation_action_apply`
5. `escalation_action_verify`
6. `read_only_guard`

## 4. Required Artifacts

- `summary.json`
- `summary.md`
- `artifact-index.json`
- per-stage logs (`*.log`)

## 5. Release Gates

Before publish:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

Publish sync:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.20 --notes-file release-notes/v0.1.20.md
make release-publish-dry VERSION=0.1.20
make release-publish VERSION=0.1.20
make release-check VERSION=0.1.20
```
