# v0.1.6 Release Gate Checklist

Date: 2026-03-08
Version: v0.1

This checklist defines the operator-facing release gate for `v0.1.6`.

## 1. Preconditions

1. API and web-console are running against a migrated database.
2. Migrations include:
   - `202603080003_create_restore_evidence_compliance_policy.sql`
   - `202603080004_create_ops_runbook_template_executions.sql`
3. At least one write-capable account and one read-only account are available.

## 2. One-Command Validation

Write-capable scenario:

```bash
bash scripts/qa-v0.1.6-operator-journey.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user admin
```

Read-only scenario:

```bash
bash scripts/qa-v0.1.6-operator-journey.sh \
  --api-base-url http://127.0.0.1:8080 \
  --auth-user viewer \
  --read-only
```

## 3. Required Stage Coverage

The summary must include these stage keys:

1. `setup_profile`
2. `next_actions`
3. `calendar_reservation`
4. `evidence_compliance`
5. `runbook_template`
6. `handover_quality`
7. `backup_evidence_writepath` (write mode may pass; read-only mode may skip)

## 4. Required Artifacts

Each run must generate:

1. `summary.json`
2. `summary.md`
3. `artifact-index.json`
4. Per-stage logs: `*.log`

Default output directory pattern:

- `.run/qa/v0.1.6-journey-<RUN_ID>/`

## 5. Failure Localization Rule

If `overall=fail`, `summary.md` must contain:

1. failed stage key,
2. endpoint path,
3. remediation pointer,
4. stage diagnostics.

Release cannot proceed until all failed stages are resolved or explicitly deferred in release closure issue.

## 6. Release Gate Commands

Before publishing:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

After release note/changelog finalization:

```bash
make release-publish-dry VERSION=0.1.6
make release-publish VERSION=0.1.6
make release-check VERSION=0.1.6
```
