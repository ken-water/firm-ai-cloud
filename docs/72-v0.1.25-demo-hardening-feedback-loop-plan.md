# CloudOps One v0.1.25 Planning (Demo Hardening + Feedback Loop)

Date: 2026-03-19
Version: v0.1
Target release: `v0.1.25`

## 1. Problem Statement

`v0.1.24` improved demo readability and scenario repeatability. Remaining near-term gaps are:

- demo narrative still depends on operator fluency instead of built-in guided flow,
- key metrics are visible but customer-value outcomes are not yet explicit,
- feedback capture from demo/pilot sessions is not productized,
- release evidence lacks standardized demo acceptance record.

## 2. Goals

1. Harden demo narrative flow from first glance to decision to action.
2. Add customer-value oriented KPI narratives in cockpit.
3. Productize pilot feedback capture and closure tracking.
4. Add demo acceptance evidence gate for release closure.

## 3. Scope

- planning and release-gate docs for `v0.1.25`,
- issue-first execution map for narrative/KPI/feedback/gate tracks,
- deterministic read-model and UI workflow for pilot feedback loop,
- validation for demo acceptance artifacts.

## 4. Out of Scope

- full multi-tenant analytics platform,
- autonomous AI execution engine,
- large-scale theming framework,
- advanced external BI connector matrix.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.25`:

1. faster customer value perception,
2. structured pilot feedback-to-roadmap loop,
3. safer operator decision support,
4. repeatable demo acceptance governance.

## 7. Definition of Done

- `#253` to `#258` closed (or explicitly deferred in release issue),
- demo narrative + KPI value + feedback loop are user-visible and validated,
- release gates pass before publish:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
  - `make release-publish-dry VERSION=0.1.25`
  - `make release-publish VERSION=0.1.25`
  - `make release-check VERSION=0.1.25`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| guided narrative oversimplifies operations reality | low trust from expert users | keep evidence links and drill paths always available |
| KPI narratives become subjective | poor adoption | tie hints to deterministic metric rules and thresholds |
| feedback capture increases operator overhead | low participation | make feedback capture lightweight with reusable templates |
| demo gate process drifts across releases | governance regression | enforce artifact checklist in release issue template |
