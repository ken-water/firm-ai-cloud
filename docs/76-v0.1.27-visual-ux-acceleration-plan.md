# CloudOps One v0.1.27 Planning (Visual System + UX Acceleration)

Date: 2026-03-20
Version: v0.1
Target release: `v0.1.27`

## 1. Problem Statement

`v0.1.26` improved role-focused cockpit flow. Remaining delivery gap to enterprise-facing demos:

- visual language is still inconsistent across sections (spacing, hierarchy, emphasis),
- first screen lacks a strong executive summary layer for non-technical stakeholders,
- large-screen experience is not yet configurable as a reusable scene set,
- AI follow-up and topology follow-up are visible but still separated from one-click dashboard composition.

## 2. Goals

1. Establish a unified cockpit visual system baseline for clearer hierarchy and stronger perceived quality.
2. Add executive summary hero blocks for fast stakeholder scanning.
3. Introduce configurable large-screen scene playlist baseline.
4. Reduce operator clicks from entry to key action paths across role scenes.

## 3. Scope

- planning and release-gate docs for `v0.1.27`,
- issue-first execution map for visual system, executive summary, scene playlist, and click-path tracks,
- UI updates in web console overview/cockpit areas,
- release validation updates for visual/scene acceptance evidence.

## 4. Out of Scope

- full design-system package extraction,
- animation-heavy marketing landing pages,
- mobile-app dedicated UX track,
- external BI embedded dashboards.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.27`:

1. stronger first impression and trust via consistent visual hierarchy,
2. faster decision making through high-signal executive summary layer,
3. practical large-screen configurability for SMB monitoring centers,
4. lower cognitive and click-path cost for daily operations.

- `#265` to `#270` are closed (or explicitly deferred in release issue),
- visual system and executive summary hero are user-visible and validated,
- scene playlist baseline is available and demoable,
- release gates pass before publish:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
  - `make release-publish-dry VERSION=0.1.27`
  - `make release-publish VERSION=0.1.27`
  - `make release-check VERSION=0.1.27`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| visual uplift causes inconsistency regressions | fragmented experience | enforce shared tokens and section-level patterns before custom blocks |
| summary hero oversimplifies technical detail | trust gap for operators | keep drill-down links and evidence references adjacent |
| scene playlist adds complexity | configuration fatigue | provide deterministic default scenes and one-click reset |
| click-path changes confuse existing users | transition friction | keep labels stable and stage changes via role-scene cues |
