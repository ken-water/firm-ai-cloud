# CloudOps One v0.0.4 UI Iteration Plan

Date: 2026-03-03

This document tracks the issue-first UI improvement process for v0.0.4.

## 1. Problem Inventory (Current State)

1. Information architecture is crowded (single-page stacked operations).
2. Visual style is inconsistent across controls and content blocks.
3. Global UX states (`loading/error/forbidden/empty`) are not consistently presented.
4. Role-based constraints exist but are not always clear in context.
5. Frontend structure is hard to evolve quickly without regressions.

## 2. Iteration Backlog

| Iteration | GitHub Issue | Focus |
| --- | --- | --- |
| I1 | `#24` | UI foundation and layout architecture |
| I2 | `#25` | CMDB assets and relation workflow UX |
| I3 | `#26` | Discovery and notification operation UX |
| I4 | `#27` | Frontend regression/quality guardrails |

## 3. Iteration Rules

1. One focused scope per iteration.
2. Keep each change set independently reversible.
3. Preserve existing backend API behavior while improving UI.
4. Keep release-facing text in English.
5. Maintain compatibility with manual-trigger CI policy.

## 4. I1 Definition of Done

- Shared design tokens are in place.
- Web-console has a stable shell layout (sidebar + main content).
- Section-level layout is visually consistent.
- Authentication/session status is visible in a dedicated topbar.
- Baseline changes compile and pass frontend build checks.
