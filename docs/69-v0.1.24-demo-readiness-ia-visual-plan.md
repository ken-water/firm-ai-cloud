# CloudOps One v0.1.24 Planning (Demo Readiness: IA + Visual Cockpit + Topology Clarity)

Date: 2026-03-19
Version: v0.1
Target release: `v0.1.24`

## 1. Problem Statement

`v0.1.23` closed copilot guided actions, topology board baseline, and handoff readiness model. Remaining customer-demo gaps are:

- module entry hierarchy is still dense and not sufficiently first-glance,
- cross-module visual language is not yet consistently demo-grade,
- topology readability still needs clearer grouping and business-path cues,
- customer walkthrough still depends on manual explanation instead of product-guided narrative.

## 2. Goals

1. Deliver a module-first information architecture entry that is immediately understandable.
2. Deliver visual cockpit baseline with coherent card hierarchy and KPI-first scanning.
3. Improve topology readability for site/business lanes and risk focus.
4. Ship a demo scenario pack so customer walkthrough is repeatable and value-oriented.

## 3. Scope

- planning and release-gate docs for `v0.1.24`,
- issue-first execution map for IA/visual/topology/demo tracks,
- cockpit UX and API read-model improvements for demo clarity,
- deterministic validation for core demo flows.

## 4. Out of Scope

- fully customizable screen studio v2,
- autonomous AI multi-step execution,
- multi-tenant org policy hierarchy redesign,
- full BI/reporting export matrix.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.24`:

1. first-glance operator comprehension,
2. business-centric visual decision cockpit,
3. topology readability for non-expert users,
4. repeatable demo-to-feedback loop.

- `#247` to `#252` closed (or explicitly deferred in release issue),
- IA/visual/topology/demo baselines are customer-demo usable,
- release gates pass before publish:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
  - `make release-publish-dry VERSION=0.1.24`
  - `make release-publish VERSION=0.1.24`
  - `make release-check VERSION=0.1.24`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| IA changes reduce discoverability for existing users | migration friction | preserve existing anchors and add compatibility links |
| visual uplift increases complexity/noise | slower operator scans | enforce KPI-first and cap visible card density |
| topology grouping may hide edge anomalies | missed risks | keep drill and raw detail paths one click away |
| demo scenario data drifts from product behavior | trust loss | generate scenarios from deterministic seed contracts |
