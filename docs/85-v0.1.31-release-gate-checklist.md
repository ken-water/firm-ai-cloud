# v0.1.31 Release Gate Checklist

Date: 2026-03-20
Version: v0.1.31

## 1. Preconditions

1. API and web-console run against migrated database.
2. `v0.1.30` release is complete and synchronized.
3. UX review baseline is aligned with:
   - `docs/84-v0.1.31-usability-ux-only-plan.md`
4. `v0.1.31` remains UX-only scope (no feature-surface expansion).

## 2. Required Planning Evidence

1. Planning issue defines only usability/UX scope.
2. IA/menu/page-focus rules are explicit and testable.
3. Acceptance metrics for entry clarity and click path are measurable.

## 3. Required Validation Coverage

1. Level-1 and level-2 navigation consistency is validated.
2. Page section tabs and default start tab behavior is validated.
3. First-screen vs advanced content separation is validated on core pages.
4. Visual consistency for typography/spacing/status semantics is validated.

## 5. Release Gates

Before publish:

```bash
cargo check --workspace
cargo test -p api
cd apps/web-console && npm run check:ui
```

Publish sync:

```bash
bash scripts/release-github-issues-check.sh --version 0.1.31 --notes-file release-notes/v0.1.31.md
make release-publish-dry VERSION=0.1.31
make release-publish VERSION=0.1.31
make release-check VERSION=0.1.31
```
