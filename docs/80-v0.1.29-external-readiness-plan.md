# CloudOps One v0.1.29 Planning (External Readiness + Pilot-to-Production Baseline)

Date: 2026-03-20
Version: v0.1
Target release: `v0.1.29`

## 1. Problem Statement

`v0.1.28` improved operator speed inside overview cockpit. Remaining gaps before larger-scale external rollout:

- first-time external users still need tighter pilot bootstrap guidance,
- demo success and production onboarding are not yet connected by one explicit handoff path,
- external trial quality signals are not standardized enough for release decisions.

## 2. Goals

1. Build external pilot bootstrap baseline for non-technical SMB operators.
2. Add demo-to-production handoff checklist with deterministic closure fields.
3. Add external readiness telemetry summary for go/no-go decision.
4. Enforce external-readiness acceptance artifacts in release closure.

## 3. Scope

- planning and release-gate docs for `v0.1.29`,
- issue-first execution map for bootstrap/handoff/readiness tracks,
- overview/setup flow enhancements for pilot onboarding,
- release validation updates for external-readiness artifacts.

## 4. Out of Scope

- full billing/subscription system,
- full marketplace and integration ecosystem,
- enterprise SSO federation hardening phase 2,
- multi-tenant data-plane isolation redesign.

## 5. Strategic Alignment

Aligned with:

- `docs/62-smb-intelligent-cloud-product-strategy-baseline.md`

Primary pillars for `v0.1.29`:

1. reduce time-to-first-value for new external pilots,
2. reduce loss between demo and real onboarding,
3. improve readiness confidence before broad external exposure,
4. tighten release governance with external-readiness evidence.

- `#277` to `#282` are closed (or explicitly deferred in release issue),
- pilot bootstrap and handoff entry are user-visible and validated,
- external readiness summary is available in overview workflow,
- release gates pass before publish:
  - `cargo check --workspace`
  - `cargo test -p api`
  - `cd apps/web-console && npm run check:ui`
  - `make release-publish-dry VERSION=0.1.29`
  - `make release-publish VERSION=0.1.29`
  - `make release-check VERSION=0.1.29`

## 8. Risks and Mitigation

| Risk | Impact | Mitigation |
| --- | --- | --- |
| bootstrap guidance too generic | low pilot conversion | keep guidance role-scoped and context-aware |
| handoff checklist becomes manual overhead | operator resistance | keep default checklist short and auto-prefilled |
| readiness summary misses key blockers | unsafe external rollout | keep explicit blocking signal with deterministic thresholds |
| release evidence misses external checks | governance regression | enforce metadata and artifact checks in gate script |
